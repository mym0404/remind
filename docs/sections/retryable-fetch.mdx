# Retryable Fetch

## 문제

실패한 요청을 같은 조건으로 다시 실행한다.

## 핵심 개념

- 실패 상태는 retry action을 보존한다.
- retry는 같은 요청 조건으로 다시 loading을 만든다.
- 성공하면 error를 지우고 data만 남긴다.

## 풀이 흐름

요청 함수를 `loadReport`로 분리한다. 실패 UI의 retry 버튼은 마지막 요청 조건으로 같은 loader를 다시 호출한다.

## 핵심 코드

```tsx
const loadReport = async () => {
  setState({ status: "loading" });
  try {
    const report = await fetchReport();
    setState({ status: "success", report });
  } catch {
    setState({ status: "error", message: "보고서를 불러오지 못했습니다." });
  }
};

{state.status === "error" ? <button onClick={loadReport}>다시 시도</button> : null}
```

## 테스트가 보는 것

- mock request의 실패 branch가 실제로 에러를 던지는지 확인한다.
- 실패 화면에서 다시 시도하면 loading을 거쳐 성공 데이터가 보이는지 본다.

## 실수 포인트

- retry에서 다른 함수를 부르면 최초 요청과 조건이 달라질 수 있다.
- error message만 있고 버튼이 없으면 사용자가 복구할 방법이 없다.
