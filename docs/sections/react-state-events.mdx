# React 상태와 이벤트

## 언제 쓰나

렌더 결과를 바꾸는 값만 `useState`에 둔다. input 값, 필터, 패널 열림, 리스트처럼 화면이 다시 그려져야 하는 값이다.

DOM focus, scroll, measurement는 `useRef`로 처리한다. 전역 이벤트는 `useEffect`에서 붙이고 cleanup에서 뗀다.

:::warning
state 변경 직후 props callback이나 외부 API에 같은 값을 넘겨야 하면 `nextTodos`처럼 다음 값을 먼저 만든다. 같은 핸들러 안에서 state 변수는 즉시 바뀌지 않는다.
:::

## 바로 쓰는 코드

```tsx title="TodoPanel.tsx"


const readFilter = (value) => {
  if (value === "active" || value === "done") {
    return value; // [!code highlight]
  }

  return "all";
};

const filterOptions = ["all", "active", "done"];

const TodoPanel = ({ initialTodos, onSave }) => {
  const [title, setTitle] = useState("");
  const [filter, setFilter] = useState("all");
  const [isOpen, setIsOpen] = useState(false);
  const [meta, setMeta] = useState({ owner: "", memo);
  // Lazy init runs only on mount.
  const [todos, setTodos] = useState(() => initialTodos); // [!code highlight]
  const titleInputRef = useRef(null);

  useEffect(() => {
    if (!isOpen) {
      return;
    }

    const handleKeyDown = (event) => {
      if (event.key === "Escape") {
        setIsOpen(false);
      }
    };

    window.addEventListener("keydown", handleKeyDown);
    return () => {
      // Remove the same listener reference.
      window.removeEventListener("keydown", handleKeyDown); // [!code highlight]
    };
  }, [isOpen]);

  const handleTitleChange = (event) => {
    setTitle(event.target.value);
  };

  const handleOwnerChange = (event) => {
    setMeta((current) => ({
      ...current,
      owner, // [!code highlight]
    }));
  };

  const handleFilterChange = (event) => {
    setFilter(readFilter(event.target.value));
  };

  const addTodo = () => {
    const nextTitle = title.trim();

    if (nextTitle.length === 0) {
      return;
    }

    // Functional update keeps rapid adds safe.
    setTodos((current) => [
      ...current,
      { id: crypto.randomUUID(), title, done,
    ]); // [!code highlight]
    setTitle("");
    titleInputRef.current?.focus();
  };

  const renameTodo = (id, nextTitle) => {
    setTodos((current) =>
      current.map((todo) =>
        todo.id === id ? { ...todo, title,
      ),
    );
  };

  const toggleTodo = (id) => {
    setTodos((current) =>
      current.map((todo) =>
        todo.id === id ? { ...todo, done, // [!code highlight]
      ),
    );
  };

  const deleteTodo = (id) => {
    setTodos((current) => current.filter((todo) => todo.id !== id));
  };

  const completeAllAndSave = () => {
    const nextTodos = todos.map((todo) => ({ ...todo, done));

    setTodos(nextTodos);
    // Share the computed next state with external code.
    onSave(nextTodos); // [!code highlight]
  };

  const handleSubmit = (event) => {
    event.preventDefault();
    onSave(todos);
    setIsOpen(false);
  };

  const visibleTodos = todos.filter((todo) => {
    if (filter === "active") {
      return !todo.done;
    }

    if (filter === "done") {
      return todo.done;
    }

    return true;
  });

  return (
    <form onSubmit={handleSubmit}>
      <button type="button" onClick={() => setIsOpen((current) => !current)}>
        {isOpen ? "닫기" : "열기"}
      </button>

      {isOpen && (
        <>
          <input value={meta.owner} onChange={handleOwnerChange} />
          <input ref={titleInputRef} value={title} onChange={handleTitleChange} />
          <button type="button" onClick={addTodo}>
            추가
          </button>

          {/* ... */}
          {filterOptions.map((option) => (
            <input
              key={option}
              type="radio"
              name="filter"
              value={option}
              checked={filter === option}
              onChange={handleFilterChange}
            />
          ))}

          {visibleTodos.map((todo) => (
            <TodoRow
              key={todo.id}
              todo={todo}
              onRename={renameTodo}
              onToggle={toggleTodo}
              onDelete={deleteTodo}
            />
          ))}

          <button type="button" onClick={completeAllAndSave}>
            모두 완료 후 저장
          </button>
          <button type="submit">저장</button>
        </>
      )}
    </form>
  );
};

// ...
```

## 실수 포인트

- 이전 state에서 다음 값을 만들 때는 setter의 함수형 업데이트를 우선한다.
- 배열과 객체 state는 새 참조로 교체한다.
- state와 callback이 같은 다음 값을 써야 하면 `nextTodos` 같은 계산 결과를 공유한다.
- DOM event listener는 cleanup에서 같은 함수 참조로 제거한다.
- `useRef`는 렌더링 데이터가 아닌 imperative 작업에만 쓴다.
