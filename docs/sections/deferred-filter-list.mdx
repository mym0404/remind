# Deferred Filter List

## 문제

입력 state는 즉시 갱신하고 무거운 필터링만 낮은 우선순위로 늦춘다.

## 핵심 개념

- input value는 transition에 넣지 않는다.
- useDeferredValue나 useTransition으로 목록 갱신을 분리한다.
- pending UI를 좁은 범위에만 보여준다.

## 풀이 흐름

input state는 즉시 갱신한다. 필터링 기준에는 `useDeferredValue(query)`를 써서 무거운 목록 갱신을 늦춘다. `query !== deferredQuery`이면 목록 영역에만 stale/pending 표시를 둔다.

## 핵심 코드

```tsx
const [query, setQuery] = useState("");
const deferredQuery = useDeferredValue(query);
const isStale = query !== deferredQuery;

const filteredMetrics = useMemo(() => {
  return metrics.filter((metric) => metric.toLowerCase().includes(deferredQuery.toLowerCase()));
}, [deferredQuery]);
```

## 테스트가 보는 것

- 긴 검색어를 입력해도 input value가 즉시 반영되는지 본다.
- 목록 갱신 상태가 별도 status로 드러나는지 확인한다.

## 실수 포인트

- input 업데이트를 transition에 넣으면 타이핑 자체가 느려진다.
- deferred 값의 dependency를 비워 두면 stale list가 고정된다.
