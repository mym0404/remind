# useReducer

## 언제 쓰나

`useReducer`는 여러 상태 조각이 한 action으로 함께 바뀔 때 쓴다. 입력값 하나나 열림 상태 하나면 `useState`가 더 낫다.

form, filter, cart, wizard, modal stack처럼 상태 도메인이 섞이면 action 이름으로 전이 규칙을 모으는 편이 읽기 쉽다.

:::warning
reducer는 순수 함수로 둔다. API 요청, 저장, analytics 전송은 이벤트 핸들러나 effect에서 실행한다.
:::

## 바로 쓰는 코드

```tsx title="Checkout.tsx"

type SortKey = "name" | "price";
type Step = "cart" | "shipping" | "review";
type ModalName = "coupon" | "help";

type Product = {
  id: string;
  name: string;
  category: string;
  price: number;
};

type CartItem = Product & {
  quantity: number;
};

type CheckoutState = {
  form: { name: string; address: string };
  filter: { query: string; category: string; sortKey: SortKey };
  cartItems: CartItem[];
  step: Step;
  modalStack: ModalName[];
};

type CheckoutAction =
  // Keep actions specific so payload narrows by type.
  | { type: "form/change"; field: keyof CheckoutState["form"]; value: string }
  | { type: "filter/change"; filter: Partial<CheckoutState["filter"]> }
  | { type: "cart/add"; product: Product } // [!code highlight]
  | { type: "cart/remove"; productId: string }
  | { type: "wizard/go"; step: Step }
  | { type: "modal/open"; name: ModalName }
  | { type: "modal/close" };

const products: Product[] = [
  // ...
];

const initialState: CheckoutState = {
  form: { name: "", address: "" },
  filter: { query: "", category: "all", sortKey: "name" },
  cartItems: [],
  step: "cart",
  modalStack: [],
};

const checkoutReducer = (
  state: CheckoutState,
  action: CheckoutAction,
): CheckoutState => {
  switch (action.type) {
    case "form/change":
      return {
        ...state,
        form: { ...state.form, [action.field]: action.value },
      };
    case "filter/change":
      return {
        ...state,
        filter: { ...state.filter, ...action.filter }, // [!code highlight]
      };
    case "cart/add": {
      const item = state.cartItems.find(
        (cartItem) => cartItem.id === action.product.id,
      );

      // Return new arrays and objects; never mutate state.
      return {
        ...state,
        cartItems: item
          ? state.cartItems.map((cartItem) =>
              cartItem.id === action.product.id
                ? { ...cartItem, quantity: cartItem.quantity + 1 } // [!code highlight]
                : cartItem,
            )
          : [...state.cartItems, { ...action.product, quantity: 1 }], // [!code highlight]
      };
    }
    case "cart/remove":
      return {
        ...state,
        cartItems: state.cartItems.filter(
          (cartItem) => cartItem.id !== action.productId,
        ),
      };
    case "wizard/go":
      return { ...state, step: action.step };
    case "modal/open":
      return { ...state, modalStack: [...state.modalStack, action.name] };
    case "modal/close":
      return { ...state, modalStack: state.modalStack.slice(0, -1) };
  }
};

const Checkout = () => {
  const [state, dispatch] = useReducer(checkoutReducer, initialState);

  const applyDrinkFilter = () => {
    dispatch({
      type: "filter/change",
      filter: { category: "drink", sortKey: "price" },
    }); // [!code highlight]
  };

  // ...

  return (
    <section>
      {/* form, filter, cart, wizard, modal UI ... */}

      <button type="button" onClick={applyDrinkFilter}>
        필터 적용
      </button>

      {products.map((product) => (
        <button
          key={product.id}
          type="button"
          onClick={() =>
            dispatch({ type: "cart/add", product }) // [!code highlight]
          }
        >
          {product.name}
        </button>
      ))}

      {/* cart and wizard controls ... */}

      {state.modalStack.at(-1) === "coupon" && ( // [!code highlight]
        <div
          role="dialog"
          aria-modal="true"
          aria-labelledby="coupon-title"
        >
          <h2 id="coupon-title">쿠폰</h2>
          <button type="button" onClick={() => dispatch({ type: "modal/close" })}>
            닫기
          </button>
        </div>
      )}
    </section>
  );
};
```

## 실수 포인트

- action type을 넓은 `string`으로 두면 payload narrowing이 깨진다.
- `push`, `splice`, 직접 대입으로 기존 상태를 바꾸지 않는다.
- 단순 setter 묶음이면 `useReducer`보다 `useState`가 낫다.
- 한 사용자 행동이 여러 setter 순서에 의존하면 reducer로 모을지 본다.
