# Stale Response Guard

## 문제

늦게 끝난 이전 요청이 최신 화면을 덮지 못하게 막는다.

## 핵심 개념

- 요청마다 최신성을 비교할 id가 필요하다.
- 늦게 끝난 이전 응답은 버린다.
- input state는 loading과 분리한다.

## 풀이 흐름

검색어 변경마다 request id를 갱신한다. 응답이 돌아오면 최신 id와 맞을 때만 결과 state에 반영한다.

## 핵심 코드

```tsx
const requestIdRef = useRef(0);

useEffect(() => {
  if (!query) return;
  const requestId = requestIdRef.current + 1;
  requestIdRef.current = requestId;
  setStatus("loading");

  searchProducts(query).then((nextResults) => {
    if (requestIdRef.current !== requestId) return;
    setResults(nextResults);
    setStatus("success");
  });
}, [query]);
```

## 테스트가 보는 것

- 느린 요청 뒤에 빠른 요청을 보내도 빠른 결과만 남는지 본다.
- loading 중에도 input value가 최신 입력을 유지하는지 확인한다.

## 실수 포인트

- 응답 순서를 요청 순서로 믿으면 race condition이 생긴다.
- loading state와 input state를 묶으면 사용자가 입력을 계속할 수 없다.
