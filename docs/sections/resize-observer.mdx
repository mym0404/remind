# ResizeObserver

## 언제 쓰나

element 자체의 크기로 UI를 맞춰야 할 때 쓴다. 사이드바 접힘, 글꼴·이미지 로딩, 부모 container 변화는 window resize 없이도 크기를 바꾼다.

floating UI에서는 trigger와 panel 크기 변화를 함께 봐야 한다. 둘 중 하나만 달라져도 top, left, placement 계산이 틀어진다.

layout state에도 쓸 수 있지만 observer callback을 `setState` 지점으로 그대로 쓰지 않는다. 크기는 callback에서 읽고, 변경 반영은 중복 비교 뒤 frame 경계로 넘긴다.

:::warning
callback 안에서 DOM read와 write를 바로 섞으면 ResizeObserver loop 경고가 날 수 있다.
:::

## 바로 쓰는 코드

```tsx title="useElementSize.ts"
const emptySize = {
  height: 0,
  width,
};

const isSameSize = (left, right) =>
  left.width === right.width && left.height === right.height;

const readBoxSize = (entry) => {
  const borderBox = entry.borderBoxSize?.[0];

  if (borderBox) {
    return {
      // Prefer border-box when padding and border affect layout.
      height: borderBox.blockSize, // [!code highlight]
      width: borderBox.inlineSize, // [!code highlight]
    };
  }

  return {
    height: entry.contentRect.height,
    width,
  };
};

const useElementSize = () => {
  const ref = useRef(null);
  const lastSizeRef = useRef(emptySize);
  const frameIdRef = useRef();
  const [size, setSize] = useState(emptySize);

  useEffect(() => {
    const element = ref.current;

    if (!element) {
      return;
    }

    const observer = new ResizeObserver((entries) => {
      const entry = entries[0];

      if (!entry) {
        return;
      }

      // Read size in the observer callback.
      const nextSize = readBoxSize(entry); // [!code highlight]

      if (isSameSize(lastSizeRef.current, nextSize)) { // [!code highlight]
        return;
      }

      if (frameIdRef.current !== undefined) {
        window.cancelAnimationFrame(frameIdRef.current);
      }

      // Defer React writes to the next frame.
      frameIdRef.current = window.requestAnimationFrame(() => {
        lastSizeRef.current = nextSize;
        setSize(nextSize); // [!code highlight]
        frameIdRef.current = undefined;
      });
    });

    observer.observe(element, { box: "border-box" }); // [!code highlight]

    return () => {
      observer.disconnect(); // [!code highlight]

      if (frameIdRef.current !== undefined) {
        window.cancelAnimationFrame(frameIdRef.current); // [!code highlight]
        frameIdRef.current = undefined;
      }
    };
  }, []);

  return { ref, size };
};
```

```tsx title="ToolbarMenu.tsx"
const getLayoutMode = (width) =>
  width < 480 ? "compact" : "regular";

const ToolbarMenu = () => {
  const { ref: containerRef, size=
    useElementSize();
  const { ref: triggerRef, size=
    useElementSize();
  const { ref: panelRef, size=
    useElementSize();

  const layoutModeRef = useRef("compact");
  const [layoutMode, setLayoutMode] = useState(layoutModeRef.current);

  useEffect(() => {
    const nextLayoutMode = getLayoutMode(containerSize.width);

    if (layoutModeRef.current === nextLayoutMode) { // [!code highlight]
      return;
    }

    layoutModeRef.current = nextLayoutMode;
    setLayoutMode(nextLayoutMode);
  }, [containerSize.width]);

  return (
    <div
      ref={containerRef}
      data-layout={layoutMode}
      style={{ position: "relative" }}
    >
      <button ref={triggerRef} type="button">
        메뉴
      </button>

      <ul
        ref={panelRef}
        style={{
          left: Math.max(0, triggerSize.width - panelSize.width), // [!code highlight]
          position: "absolute",
          top, // [!code highlight]
        }}
      >
        {/* ... */}
      </ul>
    </div>
  );
};
```

## 실수 포인트

- `contentRect`와 `borderBoxSize` 중 layout 기준에 맞는 값을 읽는다.
- 같은 크기면 `setState`하지 않는다.
- state update나 style 반영은 `requestAnimationFrame` 뒤로 넘긴다.
- floating UI는 trigger와 panel을 같이 관찰한다.
- cleanup에서 observer와 예약된 frame을 함께 정리한다.
