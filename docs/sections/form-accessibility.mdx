# Form 접근성

## 언제 쓰나

필드 이름, 도움말, 오류, 필수 여부를 입력 요소와 연결해야 할 때 쓴다. 제출 실패 후에는 첫 invalid field로 focus를 보낸다.

:::warning
오류 메시지와 `aria-invalid`는 같은 조건으로 노출한다. 둘 중 하나만 남으면 보조 기술과 화면 상태가 어긋난다.
:::

## 바로 쓰는 코드

```tsx title="ContactForm.tsx"
// ...

const describedBy = (ids) =>
  ids.filter((id): id is string => Boolean(id)).join(" ") || undefined;

const validate = (values) => {
  // ...
};

const ContactForm = () => {
  const [values, setValues] = useState({ email: "", message: "" });
  const [errors, setErrors] = useState({});
  const emailRef = useRef(null);
  const messageRef = useRef(null);
  const emailId = useId();
  const emailHelpId = useId();
  const emailErrorId = useId();
  const messageId = useId();
  const messageErrorId = useId();

  const updateEmail = ({ target }) =>
    setValues((current) => ({ ...current, email: target.value }));
  const updateMessage = ({ target }) =>
    setValues((current) => ({ ...current, message: target.value }));
  const resetForm = () => {
    // ...
  };

  const focusFirstInvalidField = (nextErrors) => {
    // Move focus after a failed submit, not while typing.
    if (nextErrors.email) {
      emailRef.current?.focus(); // [!code highlight]
      return;
    }

    if (nextErrors.message) {
      messageRef.current?.focus(); // [!code highlight]
    }
  };

  const handleSubmit = (event) => {
    event.preventDefault();

    // values come from the controlled fields.
    const nextErrors = validate(values);
    setErrors(nextErrors); // [!code highlight]

    if (Object.keys(nextErrors).length > 0) {
      focusFirstInvalidField(nextErrors);
      return;
    }

    console.log("submit", values);
  };

  return (
    <form noValidate onSubmit={handleSubmit}>
      <div>
        <label htmlFor={emailId}>이메일</label>
        <input
          id={emailId}
          ref={emailRef}
          type="email"
          value={values.email}
          onChange={updateEmail}
          required
          aria-invalid={Boolean(errors.email)} // [!code highlight]
          aria-describedby={describedBy([
            emailHelpId,
            // Only connect the error id while it is rendered.
            errors.email ? emailErrorId : undefined,
          ])} // [!code highlight]
        />
        <p id={emailHelpId}>답변을 받을 이메일을 입력하세요.</p>
        {errors.email ? (
          <p id={emailErrorId}>{errors.email}</p>
        ) : undefined}
      </div>

      {/* Repeat the same label/id/error pattern for other fields. */}
      <div>
        <label htmlFor={messageId}>문의 내용</label>
        <textarea
          id={messageId}
          ref={messageRef}
          value={values.message}
          onChange={updateMessage}
          required
          aria-invalid={Boolean(errors.message)}
          aria-describedby={errors.message ? messageErrorId : undefined}
        />
        {errors.message ? (
          <p id={messageErrorId}>{errors.message}</p>
        ) : undefined}
      </div>

      <button type="button" onClick={resetForm}>
        초기화
      </button>
      <button type="submit">보내기</button>
    </form>
  );
};
```

## 실수 포인트

- `aria-describedby`에는 현재 DOM에 있는 설명과 오류 `id`만 넣는다.
- 서버 오류나 async validation도 화면 메시지와 같은 조건으로 `aria-invalid`를 계산한다.
- 짧은 폼은 첫 invalid field, 긴 폼은 summary와 field link가 낫다.
- native form control은 `required`, `type`, `button type`을 먼저 쓴다.
- 폼 안의 명령은 clickable `div`가 아니라 `button`으로 만든다.
