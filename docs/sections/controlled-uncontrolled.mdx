# Controlled / Uncontrolled 조합

## 언제 쓰나

재사용 컴포넌트가 controlled와 uncontrolled 사용을 모두 지원해야 할 때 쓴다. Form input, Tabs, Accordion, Modal처럼 값이나 열림 상태의 주인을 호출자가 정하는 경우다.

`value`/`open`이 `undefined`가 아니면 controlled다. `defaultValue`/`defaultOpen`은 uncontrolled 초기값으로만 쓴다.

:::warning
Controlled 여부는 mount 이후 바뀌면 안 된다. `undefined`에서 문자열이나 boolean으로 바뀌는 흐름은 React 경고와 상태 소유권 변경을 만든다.
:::

## 바로 쓰는 코드

```tsx title="useControllableState.ts"

const useControllableState = <Value,>({
  value,
  defaultValue,
  onChange,
}) => {
  // `undefined` means uncontrolled; empty string and false are valid controlled values.
  const isControlled = value !== undefined; // [!code highlight]
  const [internalValue, setInternalValue] = useState(defaultValue);

  const setValue = (nextValue) => {
    if (!isControlled) {
      setInternalValue(nextValue); // [!code highlight]
    }

    onChange?.(nextValue); // [!code highlight]
  };

  return {
    // Controlled state only reports changes; uncontrolled state also stores them.
    value: isControlled ? value : internalValue,
    setValue,
  };
};
```

```tsx title="TextInput.tsx"


const TextInput = ({ value, defaultValue = "", onChange }) => {
  const input = useControllableState({
    value,
    defaultValue,
    onChange,
  });

  return (
    <input
      value={input.value}
      onChange={(event) => input.setValue(event.target.value)}
    />
  );
};
```

```tsx title="Modal.tsx"


const Modal = ({ open, defaultOpen = false, onOpenChange, children }) => {
  const modal = useControllableState({
    value: open, // [!code highlight]
    defaultValue: defaultOpen,
    onChange, // [!code highlight]
  });

  // ...

  return (
    <>
      <button type="button" onClick={() => modal.setValue(true)}>
        설정 열기
      </button>

      {modal.value ? (
        <section aria-modal="true" role="dialog">
          {children}
          <button type="button" onClick={() => modal.setValue(false)}>
            닫기
          </button>
        </section>
      ) : undefined}
    </>
  );
};
```

```tsx title="SettingsExample.tsx"
const SettingsExample = () => {
  const [name, setName] = useState("민지");
  const [isModalOpen, setIsModalOpen] = useState(false);

        {/* ... */}
      </Modal>

```

## 실수 포인트

- `value`는 `undefined`에서 문자열로 바꾸지 않는다.
- 값이 없을 수 있으면 `value={name ?? ""}`처럼 문자열을 유지한다.
- uncontrolled에서 `defaultValue`나 `defaultOpen`을 나중에 바꿔도 현재 값은 바뀌지 않는다.
- controlled 여부는 `Boolean(value)`로 판별하지 않는다.
- `value`와 `defaultValue`를 동시에 받지 않도록 props union에서 막는다.
