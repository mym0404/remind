# Toast Notifications

## 문제

toast queue, live region, 수동 닫기, 자동 제거를 한 문제로 다룬다.

## 핵심 개념

- Provider가 queue, timer, live region을 한곳에서 소유한다.
- `showToast`는 append와 auto dismiss 예약을 같이 처리한다.
- `removeToast`는 수동 닫기와 자동 제거가 공유하는 단일 제거 경로다.

## 풀이 흐름

Provider 안에 toast 배열과 timer ref를 둔다. `showToast`는 functional update로 queue에 추가하고, `removeToast`는 timer cleanup 뒤 같은 id만 제거한다.

## 핵심 코드

```tsx title="ToastProvider.tsx"
type Toast = { id: string; message: string };
type ToastContextValue = {
  showToast: (message: string) => string;
  removeToast: (id: string) => void;
};

const ToastContext = createContext<ToastContextValue | undefined>(undefined);

export const ToastProvider = ({ children }: { children: React.ReactNode }) => {
  const [toasts, setToasts] = useState<Toast[]>([]);
  const timersRef = useRef<Record<string, number>>({});

  const removeToast = useCallback((id: string) => {
    window.clearTimeout(timersRef.current[id]);
    delete timersRef.current[id];
    setToasts((current) => current.filter((toast) => toast.id !== id));
  }, []);

  const showToast = useCallback((message: string) => {
    const id = crypto.randomUUID();
    setToasts((current) => [...current, { id, message }]); // [!code highlight]
    timersRef.current[id] = window.setTimeout(() => removeToast(id), 3000);
    return id;
  }, [removeToast]);

  useEffect(() => {
    return () => Object.values(timersRef.current).forEach(window.clearTimeout); // [!code highlight]
  }, []);

  const value = useMemo(() => ({ showToast, removeToast }), [showToast, removeToast]);

  return (
    <ToastContext.Provider value={value}>
      {children}
      <div role="status" aria-live="polite">
        {toasts.map((toast) => (
          <button key={toast.id} type="button" onClick={() => removeToast(toast.id)}>
            {toast.message}
          </button>
        ))}
      </div>
    </ToastContext.Provider>
  );
};
```

## 테스트가 보는 것

- 두 번 표시하면 toast가 두 개 쌓이고 live region 안에 있는지 본다.
- 닫기 버튼과 3초 자동 제거가 같은 제거 경로로 동작하는지 확인한다.
- unmount 후 남은 timer가 cleanup되는지 본다.

## 실수 포인트

- `setToasts([...toasts, toast])`는 빠른 연속 호출에서 이전 toast를 잃을 수 있다.
- live region이 없으면 새 알림이 보조기술에 전달되지 않는다.
- timer id를 toast와 별도로 관리하지 않으면 수동 닫기 뒤 자동 제거 callback이 남는다.
