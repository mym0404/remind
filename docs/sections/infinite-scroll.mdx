# Infinite Scroll

## 언제 쓰나

- 검색 결과, 피드, 알림처럼 사용자가 계속 내려가며 보는 목록에 쓴다.
- 한 번에 전부 불러오면 응답이나 렌더링이 무거울 때 쓴다.
- footer나 하단 CTA를 반드시 봐야 하면 pagination이나 더 보기 버튼이 낫다.

## 바로 쓰는 코드

목록 끝의 sentinel을 `IntersectionObserver`로 감지한다. observer callback에서 읽는 값은 ref에 둔다.

```tsx title="FeedList.tsx"
const fetchFeedPage = async (
  page,
  signal,
) => {
  const response = await fetch(`/api/feed?page=${page}`, { signal });

  if (!response.ok) {
    throw new Error("목록을 불러오지 못했습니다.");
  }

  return response.json();
};

const FeedList = () => {
  const rootRef = useRef(null);
  const sentinelRef = useRef(null);
  const abortControllerRef = useRef(undefined);

  // Read refs inside IntersectionObserver to avoid stale state.
  const nextPageRef = useRef(1);
  const requestedPageRef = useRef(undefined);
  const isLoadingRef = useRef(false);
  const hasMoreRef = useRef(true);
  const canLoadOnIntersectRef = useRef(true); // [!code highlight]

  const [items, setItems] = useState([]);
  const [isLoading, setIsLoading] = useState(false);
  const [hasMore, setHasMore] = useState(true);
  const [errorMessage, setErrorMessage] = useState();

  const updateLoading = useCallback((value) => {
    isLoadingRef.current = value;
    setIsLoading(value);
  }, []);

  const updateHasMore = useCallback((value) => {
    hasMoreRef.current = value;
    setHasMore(value);
  }, []);

  const loadNextPage = useCallback(async () => {
    const page = nextPageRef.current;

    // The same visible sentinel can fire more than once.
    if (
      isLoadingRef.current ||
      !hasMoreRef.current ||
      requestedPageRef.current === page // [!code highlight]
    ) {
      return;
    }

    const controller = new AbortController();
    abortControllerRef.current = controller;
    requestedPageRef.current = page;
    updateLoading(true);
    setErrorMessage(undefined);

    try {
      const feedPage = await fetchFeedPage(page, controller.signal);

      if (controller.signal.aborted) {
        return;
      }

      setItems((currentItems) => [...currentItems, ...feedPage.items]);

      if (feedPage.nextPage === undefined) {
        updateHasMore(false);
        return;
      }

      nextPageRef.current = feedPage.nextPage;
    } catch (error) {
      if (controller.signal.aborted) {
        return;
      }

      requestedPageRef.current = undefined;
      setErrorMessage(
        error instanceof Error ? error.message : "알 수 없는 오류가 발생했습니다.",
      );
    } finally {
      if (abortControllerRef.current === controller) {
        abortControllerRef.current = undefined;
        updateLoading(false);
      }
    }
  }, [updateHasMore, updateLoading]);

  useEffect(() => {
    const sentinel = sentinelRef.current;

    if (!sentinel) {
      return;
    }

    const observer = new IntersectionObserver(
      ([entry]) => {
        if (!entry) return;

        if (!entry.isIntersecting) {
          canLoadOnIntersectRef.current = true;
          return;
        }

        if (!canLoadOnIntersectRef.current) return;

        canLoadOnIntersectRef.current = false; // [!code highlight]
        void loadNextPage();
      },
      {
        root: rootRef.current,
        // Prefetch before the user reaches the end.
        rootMargin: "240px 0px", // [!code highlight]
        threshold: 0.1,
      },
    );

    observer.observe(sentinel);

    return () => {
      observer.disconnect();
      abortControllerRef.current?.abort();
    };
  }, [loadNextPage]);

  const retry = () => {
    canLoadOnIntersectRef.current = true;
    void loadNextPage();
  };

  return (
    <section>
      <div ref={rootRef} style={{ maxHeight: 480, overflowY: "auto" }}>
        <ul>
          {items.map((item) => (
            <li key={item.id}>{item.title}</li>
          ))}
        </ul>
        <div ref={sentinelRef} aria-hidden="true" />
      </div>

      {isLoading && <p>불러오는 중입니다.</p>}
      {errorMessage && <button type="button" onClick={retry}>다시 시도</button>}
      {!hasMore && <p>마지막 항목입니다.</p>}
    </section>
  );
};
```

## 실수 포인트

- observer callback에서는 ref를 읽는다. `loading`, `hasMore`, page 값이 state에만 있으면 stale closure로 중복 요청이 난다.
- `requestedPageRef`로 같은 page append를 막는다.
- `hasMore`가 false면 sentinel이 보여도 더 요청하지 않는다.

:::warning
sentinel이 계속 보이면 page를 연속으로 당겨올 수 있다. 화면 밖으로 나갔다가 다시 들어올 때만 요청하도록 잠근다.
:::
