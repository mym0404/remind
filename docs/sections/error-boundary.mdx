# Error Boundary

## 언제 쓰나

렌더링 중 자식이 던진 에러를 화면 단위 fallback으로 바꿀 때 쓴다. route, page, 중요한 panel처럼 사용자가 다시 시도하거나 벗어날 수 있는 단위에 둔다.

:::warning
이벤트 핸들러, `setTimeout`, Promise rejection, `fetch` 실패는 Error Boundary가 직접 잡지 못한다.
:::

## 바로 쓰는 코드

React API는 class component에 있다.

```tsx title="ErrorBoundary.tsx"


class ErrorBoundary extends Component {
  state = { hasError: false };

  static getDerivedStateFromError() {
    // Switch rendering to fallback.
    return { hasError: true }; // [!code highlight]
  }

  componentDidCatch(error, info) {
    // Log after React builds the component stack.
    this.props.onError?.(error, info); // [!code highlight]
  }

  render() {
    if (this.state.hasError) {
      return this.props.fallback; // [!code highlight]
    }

    return this.props.children;
  }
}
```

한 번 실패한 boundary는 remount해야 초기화된다.

```tsx title="ProfileRoute.tsx"

const logRenderError = (error, info) => {
  // ...
};

const ErrorFallback = ({ onRetry }) => (
  <section role="alert">
    <h2>이 화면을 불러오지 못했습니다.</h2>
    <button type="button" onClick={onRetry}>
      다시 시도
    </button>
  </section>
);

const ProfileRoute = () => {
  const [boundaryKey, setBoundaryKey] = useState(0);

  const resetBoundary = () => {
    setBoundaryKey((key) => key + 1); // [!code highlight]
  };

  return (
    <ErrorBoundary
      // Remount to clear the failed boundary state.
      key={boundaryKey} // [!code highlight]
      fallback={<ErrorFallback onRetry={resetBoundary} />}
      onError={logRenderError}
    >
      <ProfilePanel />
      {/* ... */}
    </ErrorBoundary>
  );
};
```

## 실수 포인트

- 이벤트 핸들러와 async 실패는 호출 지점에서 처리한다.
- 앱 전체 하나도, 너무 작은 조각마다도 피한다.
- retry가 필요하면 부모 `key`로 remount한다.
- Suspense fallback과 error fallback의 의미를 섞지 않는다.
