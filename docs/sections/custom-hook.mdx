# Custom Hook

## 언제 쓰나

화면 코드보다 상태 로직이 길어질 때 custom hook으로 뺀다. 컴포넌트는 렌더링과 이벤트 연결만 남긴다.

- `useFetch`는 요청 상태와 재요청 함수를 한곳에서 관리할 때 쓴다.
- `useDebounce`는 검색어처럼 자주 바뀌는 값을 늦게 반영할 때 쓴다.
- `useLocalStorage`는 state와 `localStorage` 값을 맞춰야 할 때 쓴다.
- `useClickOutside`는 팝오버나 메뉴 바깥 클릭을 감지할 때 쓴다.
- `usePrevious`는 직전 값을 비교해야 할 때 쓴다.

좋은 hook은 특정 화면 이름, DOM 구조, CSS class를 모른다. 인자는 동작에 필요한 값만 받고, 반환값은 렌더링에 필요한 값과 함수만 담는다.

:::warning
hook API에 `options`, callback처럼 참조가 바뀌기 쉬운 값을 받으면 호출부까지 같이 설계한다. 값을 안정화하거나 primitive 인자로 좁혀 effect 재실행 조건을 예측 가능하게 둔다.
:::

## 바로 쓰는 코드

```tsx title="hooks.ts"


const useFetch = <T,>({
  enabled = true,
  options,
  url,
}) => {
  const [state, setState] = useState({
    data: undefined,
    error: undefined,
    loading: false,
  });
  const [requestCount, setRequestCount] = useState(0);

  const refetch = useCallback(() => setRequestCount((count) => count + 1), []);

  useEffect(() => {
    if (!enabled) {
      setState((current) => ({ ...current, loading));
      return;
    }

    // Cancel stale requests.
    const controller = new AbortController(); // [!code highlight]

    const load = async () => {
      setState((current) => ({ ...current, error, loading));

      try {
        const response = await fetch(url, { ...options, signal); // [!code highlight]

        if (!response.ok) {
          throw new Error(`Request failed: ${response.status}`);
        }

        const data = (await response.json()) as T;
        setState({ data, error, loading);
      } catch (error) {
        if (error instanceof DOMException && error.name === "AbortError") {
          return;
        }

        setState({
          data: undefined,
          error,
          loading,
        });
      }
    };

    void load();

    return () => {
      controller.abort(); // [!code highlight]
    };
  // Keep object deps stable.
  }, [enabled, options, requestCount, url]); // [!code highlight]

  return { ...state, refetch };
};

const useDebounce = <T,>(value, delayMs) => {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const timerId = window.setTimeout(() => setDebouncedValue(value), delayMs);

    return () => window.clearTimeout(timerId); // [!code highlight]
  }, [delayMs, value]);

  return debouncedValue;
};

const defaultDeserialize = <T,>(value) => JSON.parse(value);

const useLocalStorage = <T,>(
  key,
  initialValue,
) => {
  const [value, setValue] = useState(() => {
    if (typeof window === "undefined") {
      return initialValue;
    }

    const storedValue = window.localStorage.getItem(key);
    return storedValue === null ? initialValue : defaultDeserialize<T>(storedValue);
  });

  useEffect(() => {
    if (typeof window === "undefined") {
      return;
    }

    window.localStorage.setItem(key, JSON.stringify(value)); // [!code highlight]
  }, [key, value]); // [!code highlight]

  return [value, setValue];
};

const useClickOutside = ({
  enabled = true,
  onClickOutside,
  ref,
}) => {
  useEffect(() => {
    if (!enabled) {
      return;
    }

    const handleDocumentEvent = (event) => {
      const target = event.target;

      if (!(target instanceof Node)) {
        return;
      }

      if (ref.current && !ref.current.contains(target)) {
        onClickOutside(event); // [!code highlight]
      }
    };

    document.addEventListener("mousedown", handleDocumentEvent);
    document.addEventListener("touchstart", handleDocumentEvent);

    return () => {
      document.removeEventListener("mousedown", handleDocumentEvent);
      document.removeEventListener("touchstart", handleDocumentEvent);
    };
  // Stabilize callbacks at the call site.
  }, [enabled, onClickOutside, ref]); // [!code highlight]
};

const usePrevious = <T,>(value): T | undefined => {
  const previousRef = useRef(undefined);

  useEffect(() => {
    previousRef.current = value;
  }, [value]);

  return previousRef.current;
};
```

```tsx title="UserSearch.tsx"


const UserSearch = () => {
  const panelRef = useRef(null);
  const [isOpen, setIsOpen] = useState(false);
  const [query, setQuery] = useLocalStorage("user-search-query", "");
  const debouncedQuery = useDebounce(query, 300);
  const previousQuery = usePrevious(debouncedQuery);
  const closePanel = useCallback(() => setIsOpen(false), []);
  const url = `/api/users?query=${encodeURIComponent(debouncedQuery)}`;
  const {
    data: users = [],
    error,
    loading,
    refetch,
  } = useFetch<User[]>({
    enabled: debouncedQuery.length > 0,
    url, // [!code highlight]
  });

  useClickOutside({ enabled: isOpen, onClickOutside, ref); // [!code highlight]

  // ...

  return (
    <section ref={panelRef}>
      <input
        onFocus={() => setIsOpen(true)}
        onChange={(event) => setQuery(event.target.value)}
        value={query}
      />
      {previousQuery !== debouncedQuery && <p>검색어가 바뀌었습니다.</p>}
      {loading && <p>불러오는 중입니다.</p>}
      {error && <button onClick={refetch}>다시 시도</button>}
      {isOpen && (
        <ul>
          {users.map((user) => (
            <li key={user.id}>{user.name}</li>
          ))}
        </ul>
      )}
    </section>
  );
};
```

## 실수 포인트

- `options`, `onClickOutside` 같은 함수나 객체를 매번 새로 만들면 effect가 반복 실행된다.
- `useFetch`는 `AbortController`로 이전 요청을 정리해야 늦게 끝난 응답이 화면을 덮어쓰지 않는다.
- 반환값이 내부 상태 전이를 그대로 드러내면 호출부가 hook 내부 규칙에 묶인다.
