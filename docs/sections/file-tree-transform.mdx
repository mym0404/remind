# File Tree Transform

## 문제

flat path 배열을 임의 깊이의 folder/file tree로 바꾼다.

## 핵심 개념

- 폴더와 파일 node를 구분한다.
- 폴더를 파일보다 먼저 정렬한다.
- 같은 depth에서는 대소문자 무시 알파벳 순으로 정렬한다.

## 풀이 흐름

path를 `/`로 나눈 뒤 segment를 따라 내려가며 folder node를 만든다. 마지막 segment는 file node로 넣고 contents를 연결한다. 각 depth는 folder 우선, 그 안에서는 이름 기준으로 정렬한다.

## 핵심 코드

```tsx
const sortNodes = (nodes: TreeNode[]) =>
  [...nodes].sort((left, right) => {
    if (left.type !== right.type) return left.type === "folder" ? -1 : 1;
    return left.name.localeCompare(right.name, "en", { sensitivity: "base" });
  });

const buildFileTree = (entries: FileEntry[]) => {
  const root: TreeNode[] = [];
  // path segment를 순회하며 folder를 만들고 마지막 segment는 file로 넣는다.
  return sortNodes(root);
};
```

## 테스트가 보는 것

- 깊이가 다른 path가 중첩 tree로 바뀌는지 본다.
- folder가 file보다 먼저 오고 대소문자 차이가 정렬을 흔들지 않는지 확인한다.

## 실수 포인트

- 문자열 들여쓰기만 만들면 클릭 가능한 tree 구조를 만들기 어렵다.
- index key를 쓰면 같은 이름 정렬이나 이동에서 렌더링이 흔들린다.
