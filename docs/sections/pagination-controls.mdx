# Pagination Controls

## 문제

page state와 page size로 목록을 자르고 이전/다음 버튼 상태를 계산한다.

## 핵심 개념

- page와 pageSize에서 visible slice를 계산한다.
- prev/next는 page boundary에서 disabled다.
- pageSize 변경 후 page는 유효 범위로 clamp한다.

## 풀이 흐름

`getPageItems`, `getTotalPages`, `clampPage`를 순수 함수로 둔다. UI는 그 계산 결과만 렌더링한다.

## 핵심 코드

```tsx
const getPageItems = <Item,>(items: Item[], page: number, pageSize: number) => {
  const startIndex = (page - 1) * pageSize;
  return items.slice(startIndex, startIndex + pageSize);
};

const hasPrevious = page > 1;
const hasNext = page < totalPages;
```

## 테스트가 보는 것

- 순수 함수가 원하는 page slice를 반환하는지 본다.
- 다음 버튼을 누르면 page status와 목록 첫 항목이 바뀌는지 확인한다.

## 실수 포인트

- page가 1-base인지 0-base인지 섞이면 첫 항목이 밀린다.
- 끝 page disabled가 없으면 빈 목록으로 넘어갈 수 있다.
