# 검색 / 필터 / 정렬 리스트

## 언제 쓰나

- 원본 배열은 두고 화면 목록만 검색, 필터, 정렬할 때 쓴다.
- 조건이 바뀔 때마다 표시 목록을 다시 계산할 때 쓴다.
- 결과 없음 UI가 필요할 때 쓴다.

## 바로 쓰는 코드

원본은 그대로 두고 표시 목록만 파생한다.

```tsx title="TaskList.tsx"


const tasks = [
  {
    id: "task-1",
    title,
    owner,
    status,
    updatedAt,
  },
  // ...
];

const statusFilters = ["all", "open", "done"];
const sortKeys = ["recent", "title"];

const TaskList = () => {
  const [query, setQuery] = useState("");
  const [status, setStatus] = useState("all");
  const [sortKey, setSortKey] = useState("recent");

  const visibleTasks = useMemo(() => { // [!code highlight]
    const normalizedQuery = query.trim().toLowerCase();

    const filteredTasks = tasks.filter((task) => {
      // Keep matching cheap and local to the derived list.
      const matchesQuery =
        task.title.toLowerCase().includes(normalizedQuery) ||
        task.owner.toLowerCase().includes(normalizedQuery);
      const matchesStatus = status === "all" || task.status === status;

      return matchesQuery && matchesStatus;
    });

    // Copy before sort; sort mutates the array.
    return [...filteredTasks].sort((left, right) => { // [!code highlight]
      if (sortKey === "title") {
        return left.title.localeCompare(right.title);
      }

      return (
        new Date(right.updatedAt).getTime() -
        new Date(left.updatedAt).getTime()
      );
    });
  }, [query, status, sortKey]); // [!code highlight]

  return (
    <section>
      <label htmlFor="task-query">검색어</label>
      <input
        id="task-query"
        value={query}
        onChange={(event) => setQuery(event.target.value)}
        placeholder="제목 또는 담당자"
      />

      {/* Render filter and sort controls from statusFilters and sortKeys. */}

      {visibleTasks.length === 0 ? (
        <p>조건에 맞는 작업이 없습니다.</p>
      ) : (
        <ul>
          {visibleTasks.map((task) => (
            <li key={task.id}>{task.title}</li>
          ))}
        </ul>
      )}
    </section>
  );
};
```

:::warning
원본 배열이나 공유 배열에 `sort`를 바로 호출하면 다른 화면까지 순서가 바뀔 수 있다.
:::

## 실수 포인트

- 원본 배열을 검색 결과로 덮어쓰지 않는다.
- `sort`는 복사본에만 쓴다.
- 표시 목록은 조건을 dependency로 둔 `useMemo`에서 계산한다.
- 검색 대상이 늘어나면 정규화와 matcher를 함수로 뺀다.
- 조건이 많아지면 상태 객체나 reducer로 업데이트 경로를 줄인다.
