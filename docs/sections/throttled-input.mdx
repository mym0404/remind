# Throttled Input

## 문제

짧은 시간에 몰리는 입력 이벤트를 일정 간격으로 제한한다.

## 핵심 개념

- useThrottle hook을 만든다.
- interval 안에서는 값을 즉시 바꾸지 않는다.
- 마지막 입력은 interval 뒤에 반영한다.

## 풀이 흐름

throttle은 첫 값을 기준으로 interval을 잡고, 그 사이에 들어온 값은 trailing update 하나로 압축한다. timer ref를 두면 같은 interval 안에서 timer가 여러 개 생기는 일을 막기 쉽다.

## 핵심 코드

```tsx
const lastRunRef = useRef(0);
const trailingTimerRef = useRef<number>();

useEffect(() => {
  const remaining = interval - (Date.now() - lastRunRef.current);
  if (remaining <= 0) {
    lastRunRef.current = Date.now();
    setThrottledValue(value);
    return;
  }

  window.clearTimeout(trailingTimerRef.current);
  trailingTimerRef.current = window.setTimeout(() => setThrottledValue(value), remaining);
  return () => window.clearTimeout(trailingTimerRef.current);
}, [interval, value]);
```

## 테스트가 보는 것

- interval 안에서는 첫 값이 유지되고 마지막 값은 interval 뒤에 반영되는지 본다.
- 빠른 rerender에도 trailing timer가 하나만 남는지 확인한다.

## 실수 포인트

- debounce처럼 매번 시간을 미루면 throttle 요구와 달라진다.
- timer id를 state로 두면 불필요한 렌더링이 생긴다.
