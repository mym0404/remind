# Reducer Actions

## 문제

상태 전이를 discriminated union action으로 모은다.

## 핵심 개념

- action 하나는 상태 변화 하나만 표현한다.
- reducer는 기존 배열을 mutate하지 않는다.
- UI는 add, toggle, remove만 dispatch한다.

## 풀이 흐름

입력과 버튼 이벤트를 action으로 바꾼다. reducer는 새 배열을 반환하고 화면은 reducer state만 렌더링한다.

## 핵심 코드

```tsx
type Action =
  | { type: "add"; todo: Todo }
  | { type: "toggle"; id: string }
  | { type: "remove"; id: string };

const todoReducer = (todos: Todo[], action: Action) => {
  switch (action.type) {
    case "add": return [...todos, action.todo];
    case "toggle": return todos.map((todo) => todo.id === action.id ? { ...todo, done: !todo.done } : todo);
    case "remove": return todos.filter((todo) => todo.id !== action.id);
  }
};
```

## 테스트가 보는 것

- 이전 배열이 그대로 남는지 확인해 mutation을 잡는다.
- UI 버튼이 reducer action으로 연결되어 완료 상태를 바꾸는지 본다.

## 실수 포인트

- 배열 안 객체를 직접 바꾸면 이전 state도 함께 바뀐다.
- action payload 모양이 들쭉날쭉하면 reducer가 읽기 어려워진다.
