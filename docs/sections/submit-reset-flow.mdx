# Submit Reset Flow

## 문제

비동기 제출 중 중복 제출을 막고 성공 후 reset까지 한 흐름으로 처리한다.

## 핵심 개념

- 제출 중 submit button을 disabled 처리한다.
- 성공 메시지를 보여준다.
- 초기화 버튼은 입력값과 상태 메시지를 함께 되돌린다.

## 풀이 흐름

제출을 시작하면 `isSubmitting`을 켜고 status를 저장 중으로 바꾼다. 요청이 끝난 뒤 성공 문구를 보여주고 버튼을 다시 연다. reset은 input 값만 지우지 말고 message와 제출 상태도 초기값으로 되돌린다.

## 핵심 코드

```tsx
const [isSubmitting, setIsSubmitting] = useState(false);

const handleSubmit = async (event: FormEvent<HTMLFormElement>) => {
  event.preventDefault();
  setIsSubmitting(true);
  setMessage("저장 중");
  await saveTodo(title);
  setMessage(`${title} 저장 완료`);
  setIsSubmitting(false);
};

const resetForm = () => { setTitle(""); setMessage("대기 중"); };
```

## 테스트가 보는 것

- submit 직후 버튼 disabled와 저장 중 status를 확인한다.
- 초기화 버튼이 입력값과 status를 함께 되돌리는지 본다.

## 실수 포인트

- 요청 완료 전에 버튼을 다시 열면 중복 제출이 가능해진다.
- reset이 일부 state만 되돌리면 이전 성공 메시지가 남는다.
