# Modal Dialog

## 문제

dialog ARIA, focus 이동, Escape와 backdrop 닫기를 한 modal 문제 안에서 구현한다.

## 핵심 개념

- Provider가 현재 modal과 focus 복귀 대상을 소유한다.
- dialog는 `role`, `aria-modal`, 제목 연결을 가진다.
- open은 내부 focus, close는 trigger focus 복귀다.
- Escape와 backdrop은 같은 close 경로를 쓴다.

## 풀이 흐름

`openModal` 호출 시 현재 focus를 저장하고 modal 요청을 상태에 넣는다. Provider는 modal이 열렸을 때 닫기 버튼에 focus를 보내고, 닫힐 때 listener 정리와 focus 복귀를 처리한다.

## 핵심 코드

```tsx title="ModalProvider.tsx"
type ModalRequest = { title: string; body: React.ReactNode };
type ModalContextValue = { openModal: (request: ModalRequest) => void; closeModal: () => void };

const ModalContext = createContext<ModalContextValue | undefined>(undefined);

export const ModalProvider = ({ children }: { children: React.ReactNode }) => {
  const [modal, setModal] = useState<ModalRequest>();
  const restoreFocusRef = useRef<HTMLElement>();
  const closeButtonRef = useRef<HTMLButtonElement>(null);
  const titleId = useId();

  const closeModal = useCallback(() => {
    setModal(undefined);
    requestAnimationFrame(() => restoreFocusRef.current?.focus()); // [!code highlight]
  }, []);

  const openModal = useCallback((request: ModalRequest) => {
    const activeElement = document.activeElement;
    restoreFocusRef.current = activeElement instanceof HTMLElement ? activeElement : undefined;
    setModal(request);
  }, []);

  useEffect(() => {
    if (!modal) return;
    closeButtonRef.current?.focus();
    const onKeyDown = (event: KeyboardEvent) => event.key === "Escape" && closeModal();
    window.addEventListener("keydown", onKeyDown);
    return () => window.removeEventListener("keydown", onKeyDown); // [!code highlight]
  }, [closeModal, modal]);

  const value = useMemo(() => ({ openModal, closeModal }), [openModal, closeModal]);

  return (
    <ModalContext.Provider value={value}>
      {children}
      {modal ? (
        <div className="backdrop" onClick={(event) => event.target === event.currentTarget && closeModal()}>
          <section role="dialog" aria-modal="true" aria-labelledby={titleId}>
            <h2 id={titleId}>{modal.title}</h2>
            <div>{modal.body}</div>
            <button ref={closeButtonRef} type="button" onClick={closeModal}>닫기</button>
          </section>
        </div>
      ) : null}
    </ModalContext.Provider>
  );
};
```

## 테스트가 보는 것

- 열린 modal이 dialog role과 accessible name을 갖는지 본다.
- Escape로 닫힌 뒤 focus가 trigger로 돌아오는지 확인한다.
- backdrop click과 닫기 버튼이 같은 close 경로를 쓰는지 확인한다.

## 실수 포인트

- modal state를 각 trigger 주변에 흩어 놓으면 focus와 cleanup 경계가 중복된다.
- dialog 안쪽 클릭과 backdrop 클릭을 구분하지 않으면 내부 조작 중 modal이 닫힌다.
- keydown listener cleanup이 빠지면 닫힌 modal도 Escape에 반응한다.
