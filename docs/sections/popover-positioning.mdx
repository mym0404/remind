# Popover Positioning

## 문제

trigger와 floating rect를 읽어 기본 위치를 잡고 viewport 밖이면 보정한다.

## 핵심 개념

- trigger click으로 popover를 연다.
- outside click과 Escape로 닫는다.
- computePopoverPosition에서 flip과 shift를 처리한다.

## 풀이 흐름

trigger와 floating element의 rect를 읽어 preferred placement의 좌표를 계산한다. 아래 공간이 부족하면 위로 flip하고, 좌우로 삐져나가면 viewport 안으로 left 값을 clamp한다. 열림 상태는 trigger, outside click, Escape에서만 바꾼다.

## 핵심 코드

```tsx
const computePopoverPosition = ({
  triggerRect,
  floatingRect,
  viewportRect,
  preferredPlacement,
  offset,
}) => {
  const baseBottom = triggerRect.bottom + offset;
  const baseTop = triggerRect.top - floatingRect.height - offset;
  const hasBottomSpace = baseBottom + floatingRect.height <= viewportRect.bottom;
  const placement = hasBottomSpace ? preferredPlacement : "top";
  const top = placement === "top" ? baseTop : baseBottom;
  const left = Math.min(Math.max(triggerRect.left, viewportRect.left), viewportRect.right - floatingRect.width);

  return { top, left, placement };
};
```

## 테스트가 보는 것

- 아래 공간이 부족한 좌표에서 top placement로 flip되는지 본다.
- trigger로 열고 Escape나 outside click으로 닫히는지 확인한다.

## 실수 포인트

- DOMRect는 viewport 기준이다. absolute 좌표가 필요할 때만 scroll offset을 더한다.
- 위치 계산과 open state를 섞으면 테스트하기 어려워진다.
