# React Query / Server State

## API reference

- `queryKey`: cache identity다. `queryFn`이 읽는 id, filter, page 같은 입력은 key에도 넣는다.
- `queryFn`: 서버 데이터를 반환하는 async 함수다. 실패는 `throw`로 드러낸다.
- loading/error/data state: 로딩은 `isPending`, 오류는 `isError`와 `error`, 성공 데이터는 `data`로 분기한다.
- `invalidateQueries`: matching query를 stale로 만들고, 활성 query가 있으면 다시 가져오게 한다.
- `staleTime`: 데이터를 fresh로 보는 시간이다. 이 시간 안에서는 mount, focus 같은 refetch trigger가 와도 새 요청을 줄일 수 있다.

## 쓰는 곳

서버가 원본이고 화면 밖에서도 바뀔 수 있는 값에 쓴다. Todo 목록, 사용자 정보, 결제 내역처럼 네트워크로 읽는 값이 대표적이다.

입력값, 열린 modal, 선택된 tab처럼 브라우저 안에서만 의미가 있는 값은 `useState`나 `useReducer`로 충분하다.

React Query는 cache, refetch, loading/error/data state, mutation 후 동기화를 한 흐름으로 묶고 싶을 때 쓴다.

## 바로 쓰는 코드

```tsx title="todos-query.tsx"
const todoKeys = {
  all: ["todos"] as const,
  list: (status: string) => [...todoKeys.all, { status }] as const,
};
const Todos = ({ status }: { status: string }) => {
  const queryClient = useQueryClient();
  const queryKey = todoKeys.list(status);
  const todosQuery = useQuery({
    queryKey,
    queryFn: async () => {
      const response = await fetch(`/api/todos?status=${status}`);
      if (!response.ok) throw new Error("Failed to load todos");
      return (await response.json()) as Todo[];
    },
    staleTime: 30_000,
  });

  const createTodo = useMutation({
    mutationFn: async (title: string) => {
      const response = await fetch("/api/todos", {
        method: "POST",
        body: JSON.stringify({ title }),
      });
      if (!response.ok) throw new Error("Failed to create todo");
    },
    onSuccess: () => {
      void queryClient.invalidateQueries({ queryKey: todoKeys.all });
    },
  });

  if (todosQuery.isPending) return <p>불러오는 중</p>;
  if (todosQuery.isError) return <p>{todosQuery.error.message}</p>;
  return <button onClick={() => createTodo.mutate("새 할 일")}>{todosQuery.data.length}개</button>;
};
```

## 실수 포인트

- query key에 입력값을 빼면 cache가 섞인다.
- 서버 state를 local state로 복사해 오래 들고 있지 않는다.
- mutation 성공 후 관련 query를 invalidate하지 않으면 stale data가 남는다.
- `staleTime`을 길게 잡으면 refetch는 줄지만 사용자가 오래된 데이터를 볼 수 있다.
