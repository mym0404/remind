# Optimistic Update

## 언제 쓰나

좋아요, 저장, 삭제, 댓글 작성처럼 결과를 바로 기대하는 액션에 쓴다. 먼저 화면을 바꾸고, 성공하면 서버 응답으로 확정하고, 실패하면 되돌린다.

React 19에서는 `useOptimistic`으로 pending 동안만 보이는 임시 상태를 만들 수 있다. 직접 구현할 때는 이전 값 보관과 같은 항목의 중복 요청 차단이 핵심이다.

## 바로 쓰는 코드

```tsx title="CommentForm.tsx"
const createComment = async (text) => {
  const response = await fetch("/api/comments", {
    body: JSON.stringify({ text }),
    method: "POST",
    // ...
  });

  if (!response.ok) {
    throw new Error("댓글을 저장하지 못했습니다.");
  }

  return (await response.json()) as Comment;
};

const CommentForm = ({
  initialComments,
}) => {
  const formRef = useRef(null);
  const isSubmittingRef = useRef(false);
  const [comments, setComments] = useState(initialComments);
  const [errorMessage, setErrorMessage] = useState();
  const [optimisticComments, addOptimisticComment] = useOptimistic(
    comments,
    // Keep this function pure.
    (currentComments, comment) => [...currentComments, comment], // [!code highlight]
  );

  const formAction = async (formData) => {
    if (isSubmittingRef.current) {
      return;
    }

    const text = String(formData.get("comment") ?? "").trim();

    if (!text) {
      return;
    }

    isSubmittingRef.current = true;
    setErrorMessage(undefined);
    // Visible only while the action is pending.
    addOptimisticComment({
      id: crypto.randomUUID(),
      status: "saving",
      text,
    }); // [!code highlight]
    formRef.current?.reset();

    try {
      const savedComment = await createComment(text);
      // Commit the server shape, not the optimistic shape.
      setComments((currentComments) => [...currentComments, savedComment]); // [!code highlight]
    } catch (error) {
      setErrorMessage(
        error instanceof Error ? error.message : "댓글 저장 중 문제가 생겼습니다.",
      );
    } finally {
      isSubmittingRef.current = false;
    }
  };

  return (
    <section>
      {/* ... */}

      <form action={formAction} ref={formRef}>
        {/* ... */}
        <input name="comment" />
        <button type="submit">등록</button>
      </form>

      {errorMessage && <p role="alert">{errorMessage}</p>}
    </section>
  );
};
```

직접 구현할 때는 바뀐 항목만 되돌릴 수 있어야 한다.

:::warning
Rollback을 전체 목록 snapshot으로 처리하면 그 사이 성공한 다른 요청까지 덮어쓸 수 있다.
:::

```tsx title="TodoList.tsx"
const updateTodoDone = async ({
  done,
  id,
}) => {
  const response = await fetch(`/api/todos/${id}`, {
    body: JSON.stringify({ done }),
    method: "PATCH",
    // ...
  });

  if (!response.ok) {
    throw new Error("할 일을 저장하지 못했습니다.");
  }

  return (await response.json()) as Todo;
};

const TodoList = ({
  initialTodos,
}) => {
  const pendingIdsRef = useRef({});
  const [todos, setTodos] = useState(initialTodos);
  const [pendingIds, setPendingIds] = useState({});
  const [errorMessage, setErrorMessage] = useState();

  const setTodoPending = (id, isPending) => {
    const nextPendingIds = { ...pendingIdsRef.current };

    if (isPending) {
      nextPendingIds[id] = true;
    } else {
      delete nextPendingIds[id];
    }

    pendingIdsRef.current = nextPendingIds;
    setPendingIds(nextPendingIds);
  };

  const toggleTodo = async (todo) => {
    if (pendingIdsRef.current[todo.id]) {
      // Ignore duplicate writes for the same item.
      return; // [!code highlight]
    }

    const previousTodo = todo;
    const nextDone = !todo.done;

    setTodoPending(todo.id, true);
    setErrorMessage(undefined);
    // Optimistically patch only the touched item.
    setTodos((currentTodos) =>
      currentTodos.map((item) =>
        item.id === todo.id ? { ...item, done: nextDone } : item,
      ),
    ); // [!code highlight]

    try {
      const savedTodo = await updateTodoDone({
        done: nextDone,
        id: todo.id,
      });

      // Server response may include id, sorting, or permission changes.
      setTodos((currentTodos) =>
        currentTodos.map((item) =>
          item.id === savedTodo.id ? savedTodo : item,
        ),
      ); // [!code highlight]
    } catch (error) {
      const message =
        error instanceof Error ? error.message : "할 일 저장 중 문제가 생겼습니다.";

      // Roll back the item touched by this request.
      setTodos((currentTodos) =>
        currentTodos.map((item) =>
          item.id === previousTodo.id ? previousTodo : item,
        ),
      ); // [!code highlight]
      setErrorMessage(message);
    } finally {
      setTodoPending(todo.id, false);
    }
  };

  return (
    <section>
      {/* ... */}

      {errorMessage && <p role="alert">{errorMessage}</p>}
    </section>
  );
};
```

## 실수 포인트

- optimistic 값만 남기고 서버 응답을 버리면 `id`, 정렬, 권한 결과가 어긋난다.
- 같은 항목의 중복 요청을 허용하면 빠른 연속 클릭으로 상태가 꼬인다.
- `useOptimistic` update 함수 안에서 `crypto.randomUUID()` 같은 부수 효과를 만들지 않는다.
- rollback 기준은 전체 목록이 아니라 이번 요청이 건드린 항목이다.
