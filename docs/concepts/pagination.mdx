# Pagination

## API reference

Pagination은 보통 `page`와 `pageSize`를 상태로 둔다. `page`는 1부터 시작하게 두면 화면 표시와 계산이 단순하다.

```tsx title="pagination-state.tsx"
const [page, setPage] = useState(1);
const [pageSize, setPageSize] = useState(20);
```

이미 받은 배열은 `slice(start, end)`로 자른다. `end`는 포함되지 않으므로 `start + pageSize`를 그대로 넣는다.

```ts title="pagination-range.ts"
type PageRange = {
  start: number;
  end: number;
};

const getPageRange = ({
  page,
  pageSize,
}: {
  page: number;
  pageSize: number;
}): PageRange => {
  const start = (page - 1) * pageSize;

  return {
    start,
    end: start + pageSize,
  };
};

const { start, end } = getPageRange({ page, pageSize });
const visibleItems = items.slice(start, end);
```

전체 개수를 알고 있으면 page 수와 이동 가능 여부를 helper로 묶어 계산한다.

```ts title="pagination-meta.ts"
type PaginationMeta = {
  currentPage: number;
  pageCount: number;
  hasPrev: boolean;
  hasNext: boolean;
};

const getPaginationMeta = ({
  page,
  pageSize,
  totalCount,
}: {
  page: number;
  pageSize: number;
  totalCount: number;
}): PaginationMeta => {
  const pageCount = Math.max(1, Math.ceil(totalCount / pageSize));
  const currentPage = Math.min(Math.max(page, 1), pageCount);

  return {
    currentPage,
    pageCount,
    hasPrev: currentPage > 1,
    hasNext: currentPage < pageCount,
  };
};
```

공유, 새로고침, 뒤로 가기가 필요하면 `page`와 `pageSize`를 search param에 둔다. 이때 `URLSearchParams`에는 문자열을 넣는다.

```ts title="pagination-params.ts"
const params = new URLSearchParams({
  page: String(page),
  pageSize: String(pageSize),
});

const nextUrl = `/orders?${params.toString()}`;
```

이전/다음 버튼은 `disabled`를 props로 넘긴다. 서버 목록에서는 loading 중에도 막아야 중복 요청이 줄어든다.

```tsx title="pagination-buttons.tsx"
<>
  <button type="button" onClick={() => setPage(page - 1)} disabled={!hasPrev || isLoading}>
    이전
  </button>
  <button type="button" onClick={() => setPage(page + 1)} disabled={!hasNext || isLoading}>
    다음
  </button>
</>
```

페이지 번호를 link로 보여줄 때는 현재 page에 `aria-current="page"`를 둔다.

```tsx title="pagination-links.tsx"
{pageNumbers.map((pageNumber) => (
  <a
    key={pageNumber}
    href={`?${new URLSearchParams({
      page: String(pageNumber),
      pageSize: String(pageSize),
    })}`}
    aria-current={pageNumber === page ? "page" : undefined}
  >
    {pageNumber}
  </a>
))}
```

## 언제 쓰나

긴 목록을 page 단위로 끊어 보여줄 때 쓴다. 검색 결과, 주문 목록, 관리자 테이블처럼 위치를 다시 찾아야 하는 화면에 잘 맞는다.

이미 받은 배열은 `page`와 `pageSize`로 `slice`한다. 서버 목록은 `page`, `pageSize`를 보내고 현재 page의 데이터와 `hasNext`를 받는다.

## 바로 쓰는 코드

클라이언트 배열은 현재 page를 보정한 뒤 자른다.

```tsx title="LocalOrderList.tsx"
const pageSize = 10;

const LocalOrderList = ({ orders }) => {
  const [page, setPage] = useState(1);
  const pageCount = Math.max(1, Math.ceil(orders.length / pageSize));
  // Clamp after filters shrink the list.
  const currentPage = Math.min(page, pageCount); // [!code highlight]

  const visibleOrders = useMemo(() => {
    const startIndex = (currentPage - 1) * pageSize;

    return orders.slice(startIndex, startIndex + pageSize);
  }, [currentPage, orders]);

  const hasPrev = currentPage > 1;
  const hasNext = currentPage < pageCount;

  return (
    <section>
      <ul>
        {visibleOrders.map((order) => (
          <li key={order.id}>{order.title}</li>
        ))}
      </ul>

      <nav aria-label="주문 목록 페이지">
        <button type="button" onClick={() => setPage(currentPage - 1)} disabled={!hasPrev}>
          이전
        </button>
        <span>
          {currentPage} / {pageCount}
        </span>
        <button type="button" onClick={() => setPage(currentPage + 1)} disabled={!hasNext}>
          다음
        </button>
      </nav>
    </section>
  );
};
```

서버 pagination은 page 변경마다 다시 요청한다. 중복 요청과 stale response만 막으면 된다.

```tsx title="ServerOrderList.tsx"
const pageSize = 20;

const fetchOrdersPage = async ({
  page,
  signal,
}) => {
  const params = new URLSearchParams({
    page: String(page),
    pageSize: String(pageSize),
  });
  const response = await fetch(`/api/orders?${params.toString()}`, { signal });

  if (!response.ok) {
    throw new Error("주문 목록을 불러오지 못했습니다.");
  }

  const body = await response.json();

  return body;
};

const ServerOrderList = () => {
  const [page, setPage] = useState(1);
  const [orders, setOrders] = useState([]);
  const [hasNext, setHasNext] = useState(false);
  const [isLoading, setIsLoading] = useState(false);
  const [errorMessage, setErrorMessage] = useState();
  const listTopRef = useRef(null);
  const isLoadingRef = useRef(false);

  useEffect(() => {
    const controller = new AbortController();

    const loadOrders = async () => {
      isLoadingRef.current = true;
      setIsLoading(true);
      setErrorMessage(undefined);

      try {
        const body = await fetchOrdersPage({
          page,
          // Abort prevents older responses from winning the race.
          signal: controller.signal, // [!code highlight]
        });

        setOrders(body.items);
        setHasNext(body.hasNext); // [!code highlight]
        // Keep the user near the new list start.
        listTopRef.current?.scrollIntoView({ block: "start" }); // [!code highlight]
      } catch (error) {
        if (controller.signal.aborted) {
          return;
        }

        // ...
      } finally {
        if (!controller.signal.aborted) {
          isLoadingRef.current = false;
          setIsLoading(false);
        }
      }
    };

    void loadOrders();

    return () => {
      controller.abort();
    };
  }, [page]);

  const hasPrev = page > 1;

  const changePage = (nextPage) => {
    // Do not rely only on disabled buttons.
    const cannotChangePage =
      isLoadingRef.current || nextPage < 1 || nextPage === page; // [!code highlight]

    if (cannotChangePage) {
      return;
    }

    isLoadingRef.current = true;
    setIsLoading(true);
    setPage(nextPage);
  };

  return (
    <section>
      <div ref={listTopRef} />

      {isLoading && <p>불러오는 중입니다.</p>}
      {errorMessage && <p role="alert">{errorMessage}</p>}

      <ul>
        {orders.map((order) => (
          <li key={order.id}>{order.title}</li>
        ))}
      </ul>

      <nav aria-label="주문 목록 페이지">
        <button
          type="button"
          onClick={() => changePage(page - 1)}
          disabled={!hasPrev || isLoading}
        >
          이전
        </button>
        <span>{page}</span>
        <button
          type="button"
          onClick={() => changePage(page + 1)}
          disabled={!hasNext || isLoading}
        >
          다음
        </button>
      </nav>
    </section>
  );
};
```

## 실수 포인트

- filter나 search 후 `pageCount`가 줄면 현재 page를 보정한다.
- total count가 없으면 `hasNext`를 응답 contract로 둔다.
- loading button과 handler 양쪽에서 중복 요청을 막는다.
- fetch cleanup에서 이전 요청을 abort한다.
- 공유와 뒤로 가기가 필요하면 `page`를 search param에 둔다.
