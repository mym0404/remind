# Pagination

## API reference

- `page`: 사용자가 보고 있는 1-based page 번호다.
- `pageSize`: 한 page에 보여줄 item 수다.
- `totalPages`: 전체 item 수를 `pageSize`로 나눈 뒤 올림하고, 최소 `1`로 둔다.
- `slice`: `(page - 1) * pageSize`부터 `start + pageSize` 전까지 자른다.
- `clamp`: `page`를 `1`과 `totalPages` 사이로 보정한다.
- `disabled`: 이전/다음 버튼이 경계를 넘거나 loading 중일 때 이동을 막는다.

## 쓰는 곳

검색 결과, 주문 목록, 관리자 테이블처럼 긴 목록에서 현재 위치를 page 단위로 다시 찾아야 할 때 쓴다.

이미 받은 배열은 `slice`로 자르고, 서버 목록은 `page`, `pageSize`를 요청 contract로 둔다.

## 바로 쓰는 코드

```tsx title="pagination.tsx"
type PaginationInput<T> = { items: T[]; page: number; pageSize: number };

const clamp = (page: number, totalPages: number) =>
  Math.min(Math.max(page, 1), totalPages);

const paginate = <T,>({ items, page, pageSize }: PaginationInput<T>) => {
  const totalPages = Math.max(1, Math.ceil(items.length / pageSize));
  const currentPage = clamp(page, totalPages);
  const start = (currentPage - 1) * pageSize;

  return {
    currentPage,
    totalPages,
    items: items.slice(start, start + pageSize),
    hasPrev: currentPage > 1,
    hasNext: currentPage < totalPages,
  };
};

const pageInfo = paginate({ items, page, pageSize });
const changePage = (nextPage: number) => {
  if (nextPage < 1 || nextPage > pageInfo.totalPages) return;
  setPage(nextPage);
};

<nav aria-label="페이지 이동">
  <button type="button" onClick={() => changePage(pageInfo.currentPage - 1)} disabled={!pageInfo.hasPrev}>
    이전
  </button>
  <span>{`${pageInfo.currentPage} / ${pageInfo.totalPages}`}</span>
  <button type="button" onClick={() => changePage(pageInfo.currentPage + 1)} disabled={!pageInfo.hasNext}>
    다음
  </button>
</nav>;
```

## 실수 포인트

- `page`는 화면과 URL에 맞춰 1-based로 두고, 배열 index 계산에서만 0-based로 바꾼다.
- filter나 search 후 `totalPages`가 줄면 현재 `page`를 `clamp`한다.
- 이전/다음 이동은 `disabled`만 믿지 말고 handler에서도 경계를 확인한다.
- 서버 목록은 loading 중에도 버튼을 막아 중복 요청을 줄인다.
