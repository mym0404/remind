# useDebounce

## API reference

- `setTimeout`: `delay` 뒤에 최신 `value`를 debounced state로 반영한다.
- `clearTimeout`: `value`나 `delay`가 다시 바뀌기 전의 예약을 지운다.
- `useEffect` deps: `[value, delay]`를 넣어 입력값과 지연 시간이 바뀔 때 timer를 다시 만든다.
- returned debounced value: 즉시 바뀌는 원본 값이 아니라 마지막 변경 후 `delay`가 지난 값이다.

## 쓰는 곳

- 검색 요청, 자동 저장, resize 계산처럼 짧은 시간에 여러 번 바뀌는 값에서 마지막 변경만 반영할 때 쓴다.

## 바로 쓰는 코드

```tsx title="useDebounce.ts"
import { useEffect, useState } from "react";

export const useDebounce = <T,>(value: T, delay: number): T => {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const timerId = window.setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      window.clearTimeout(timerId);
    };
  }, [value, delay]);

  return debouncedValue;
};
```

## 실수 포인트

- input의 원본 state까지 debounce하면 타이핑 반응이 늦어진다.
- `delay`를 deps에서 빼면 지연 시간이 바뀌어도 timer가 갱신되지 않는다.
- cleanup이 없으면 이전 timer가 나중에 실행되어 오래된 값을 반영할 수 있다.
