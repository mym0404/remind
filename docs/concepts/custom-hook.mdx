# Custom Hook

## API reference

- 이름은 `use`로 시작한다.
- hook은 component나 다른 hook의 최상위에서만 호출한다.
- 인자는 hook이 제어할 외부 조건만 받는다.
- 반환값은 호출부가 바로 쓰는 값, action, ref로 제한한다.
- timer, listener, observer, request를 만들면 cleanup도 hook 안에 둔다.

## 쓰는 곳

- 여러 컴포넌트가 같은 state/effect 규칙을 공유할 때 쓴다.
- 컴포넌트보다 timer, listener, request 정리 코드가 더 눈에 띌 때 쓴다.
- 화면 이름, DOM 구조, CSS class를 모르는 behavior만 hook으로 뺀다.

## 바로 쓰는 코드

```tsx title="hooks.ts"
const useDebouncedValue = <T,>(value: T, delayMs: number) => {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const timerId = window.setTimeout(() => {
      setDebouncedValue(value);
    }, delayMs);

    return () => window.clearTimeout(timerId); // [!code highlight]
  }, [delayMs, value]);

  return debouncedValue;
};

const useDocumentListener = <K extends keyof DocumentEventMap>({
  enabled = true,
  type,
  onEvent,
}: {
  enabled?: boolean;
  type: K;
  onEvent: (event: DocumentEventMap[K]) => void;
}) => {
  useEffect(() => {
    if (!enabled) return;

    document.addEventListener(type, onEvent);

    return () => {
      document.removeEventListener(type, onEvent); // [!code highlight]
    };
  }, [enabled, onEvent, type]);
};
```

## 실수 포인트

- 조건문, 반복문, callback 안에서 hook을 호출하지 않는다.
- 한 곳에서만 쓰는 단순 state를 hook으로 빼지 않는다.
- 객체나 함수를 인자로 받으면 호출부의 참조 안정성까지 같이 본다.
- hook 반환 API를 과하게 넓히지 않는다.
