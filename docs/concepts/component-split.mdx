# 컴포넌트 분리

## API reference

- state owner: 값을 만들고 바꾸는 컴포넌트다. state 전환 규칙도 여기에 둔다.
- presentational component: props를 받아 화면만 그린다. 데이터 조회나 parent state 규칙을 모른다.
- callback prop: child에서 생긴 이벤트를 owner로 올리는 함수다. 이름은 `onToggle`, `onValueChange`처럼 화면 행동 기준으로 둔다.
- derived value: props나 state에서 계산한 값이다. 같은 값을 별도 state로 복사하지 않는다.

## 쓰는 곳

- state 변경 코드와 JSX가 서로 읽기 어렵게 만들 때 쓴다.
- 같은 UI 조각이 반복되거나 이름 붙일 만큼 의미가 있을 때 쓴다.
- 여러 형제 컴포넌트가 같은 값이나 이벤트를 함께 쓸 때 쓴다.
- 필터링, 정렬, 집계 같은 계산이 render 흐름을 가릴 때 쓴다.

## 바로 쓰는 코드

```tsx title="split-boundary.tsx"
const TodoPage = () => {
  const [todos, setTodos] = useState(initialTodos);

  const doneCount = todos.filter((todo) => todo.done).length;

  const toggleTodo = (id: string) => {
    setTodos((current) =>
      current.map((todo) =>
        todo.id === id ? { ...todo, done: !todo.done } : todo,
      ),
    );
  };

  return (
    <section>
      <p>완료 {doneCount}</p>
      <TodoList todos={todos} onToggle={toggleTodo} />
    </section>
  );
};

const TodoList = ({ todos, onToggle }) => (
  <ul>
    {todos.map((todo) => (
      <TodoItem key={todo.id} todo={todo} onToggle={onToggle} />
    ))}
  </ul>
);
```

`TodoPage`는 state owner다. `TodoList`는 받은 값과 이벤트만 아래로 넘긴다. `doneCount`는 state가 아니라 derived value로 둔다.

## 실수 포인트

- 작게 보인다는 이유만으로 파일부터 나누지 않는다.
- child가 parent state 구조를 알아야 하면 경계가 잘못된 것이다.
- 같은 값을 state 여러 개로 복사하지 않는다.
- 한 번만 쓰는 wrapper는 만들지 않는다.
- callback prop을 `handleClick`처럼 내부 구현 이름으로 넘기지 않는다.
