# Context + Provider

## API reference

- `createContext(defaultValue)`: Context 객체를 만든다. Provider가 없을 때만 `defaultValue`가 읽힌다.
- `<Context.Provider value={value}>`: 하위 tree에 값을 공급한다. 하위 컴포넌트는 가장 가까운 Provider의 `value`를 읽는다.
- `useContext(Context)`: 현재 컴포넌트에서 Context 값을 읽는다. Provider가 없으면 `createContext`의 기본값을 반환한다.
- guard hook: 기본값을 `undefined`로 두고, custom hook에서 Provider 누락을 바로 에러로 만든다.
- `useMemo`: Provider의 `value`가 객체일 때 참조를 안정화한다. 의존성이 같으면 이전 객체를 재사용한다.

## 쓰는 곳

- Theme, auth user, locale처럼 여러 깊이의 컴포넌트가 같은 앱 상태를 읽을 때 쓴다.
- Toast, modal처럼 화면 어디서든 같은 action을 호출해야 할 때 쓴다.
- 한두 단계 전달이면 Context보다 props가 더 단순하다.

## 바로 쓰는 코드

```tsx title="ThemeProvider.tsx"
type Theme = "light" | "dark";

type ThemeContextValue = {
  theme: Theme;
  setTheme: (theme: Theme) => void;
};

const ThemeContext = createContext<ThemeContextValue | undefined>(undefined);

const ThemeProvider = ({ children }: { children: React.ReactNode }) => {
  const [theme, setTheme] = useState<Theme>("light");
  const value = useMemo(() => ({ theme, setTheme }), [theme]);

  return <ThemeContext.Provider value={value}>{children}</ThemeContext.Provider>;
};

const useTheme = () => {
  const context = useContext(ThemeContext);

  if (context === undefined) {
    throw new Error("useTheme must be used within ThemeProvider");
  }

  return context;
};
```

## 실수 포인트

- Context 기본값에 더미 객체를 넣으면 Provider 누락이 숨는다.
- `value={{ theme, setTheme }}`처럼 새 객체를 바로 넘기면 구독 컴포넌트가 불필요하게 다시 렌더링될 수 있다.
- 자주 바뀌는 큰 객체를 Context에 통째로 넣으면 작은 변경도 넓게 퍼진다.
- Provider를 앱 전체에 무조건 두면 렌더링과 상태 소유 범위가 커진다.
