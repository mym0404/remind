# Modal

## API reference

- `role="dialog"`: 현재 화면 위에 뜬 대화 상자임을 알린다.
- `aria-modal="true"`: dialog가 열린 동안 뒤쪽 화면을 다루지 않는 상태임을 알린다.
- `aria-labelledby`: dialog 제목의 `id`를 연결한다.
- focus 이동/복귀: 열릴 때 내부 조작 요소로 보내고, 닫힐 때 trigger로 돌린다.
- Escape/backdrop: 사용자가 닫기를 기대하는 두 경로를 같은 `onClose`로 묶는다.

## 쓰는 곳

- 삭제 확인처럼 현재 흐름보다 먼저 결정해야 하는 작업에 쓴다.
- 짧은 입력이나 설정 변경처럼 화면 뒤 맥락을 유지한 채 한 가지 일을 끝낼 때 쓴다.
- 단순 안내는 modal보다 inline message나 toast가 덜 방해된다.

## 바로 쓰는 코드

Provider가 현재 modal 요청과 focus 복귀 대상을 소유한다. 화면 어디서든 `useModal().openModal()`로 열 수 있다.

```tsx title="ModalProvider.tsx"
type ModalRequest = { title: string; content: React.ReactNode; trigger?: HTMLElement };
type ModalContextValue = {
  openModal: (request: ModalRequest) => void;
  closeModal: () => void;
};

const ModalContext = createContext<ModalContextValue | undefined>(undefined);

export const ModalProvider = ({ children }: { children: React.ReactNode }) => {
  const [modal, setModal] = useState<ModalRequest>();
  const closeButtonRef = useRef<HTMLButtonElement>(null);
  const restoreFocusRef = useRef<HTMLElement>();
  const titleId = useId();

  const closeModal = useCallback(() => {
    setModal(undefined);
    requestAnimationFrame(() => restoreFocusRef.current?.focus()); // [!code highlight]
  }, []);

  const openModal = useCallback((request: ModalRequest) => {
    const activeElement = document.activeElement;
    restoreFocusRef.current = request.trigger ?? (activeElement instanceof HTMLElement ? activeElement : undefined);
    setModal(request);
  }, []);

  useEffect(() => {
    if (!modal) return;

    closeButtonRef.current?.focus(); // [!code highlight]

    const handleKeyDown = (event) => {
      if (event.key === "Escape") closeModal();
    };

    window.addEventListener("keydown", handleKeyDown);
    return () => window.removeEventListener("keydown", handleKeyDown); // [!code highlight]
  }, [closeModal, modal]);

  const value = useMemo(() => ({ openModal, closeModal }), [openModal, closeModal]);

  return (
    <ModalContext.Provider value={value}>
      {children}
      {modal ? (
        <div className="backdrop" onClick={(event) => event.target === event.currentTarget && closeModal()}>
          <section aria-labelledby={titleId} aria-modal="true" role="dialog">
            <h2 id={titleId}>{modal.title}</h2>
            <div>{modal.content}</div>
            <button ref={closeButtonRef} type="button" onClick={closeModal}>닫기</button>
          </section>
        </div>
      ) : null}
    </ModalContext.Provider>
  );
};

export const useModal = () => {
  const context = useContext(ModalContext);
  if (context === undefined) throw new Error("useModal must be used within ModalProvider");
  return context;
};
```

## 실수 포인트

- modal state를 각 버튼 주변에 흩어 놓으면 focus 복귀와 Escape cleanup이 중복된다.
- 제목 없는 dialog를 만들지 않는다.
- dialog 안쪽 클릭을 backdrop click으로 처리하지 않는다.
- Escape listener를 닫힌 뒤에도 남기지 않는다.
- 닫힌 뒤 focus 복귀 대상을 잃지 않는다.
