# 이벤트 버블링 / 캡처링 / stopPropagation

## API reference

| API | 핵심 |
|---|---|
| capture / bubble | capture는 root에서 target으로 내려가고, bubble은 target에서 root로 올라간다. React는 `onClickCapture`, `onClick`처럼 phase를 handler 이름으로 나눈다. |
| `event.target` / `event.currentTarget` | `target`은 실제 이벤트가 시작된 element이고, `currentTarget`은 지금 실행 중인 handler가 붙은 element다. |
| `event.stopPropagation()` / `event.preventDefault()` | `stopPropagation`은 남은 전파를 막고, `preventDefault`는 submit, link 이동 같은 브라우저 기본 동작만 막는다. |

## 쓰는 곳

- Backdrop 클릭으로 dialog를 닫되 dialog 내부 클릭은 무시할 때 쓴다.
- Card, menu, table row 안쪽에 별도 button이나 link가 있을 때 쓴다.
- Capture phase는 로깅이나 바깥 감시에만 좁게 둔다.

## 바로 쓰는 코드

Backdrop은 내부 element마다 `stopPropagation`을 붙이기보다 `target`과 `currentTarget`으로 경계를 확인하는 편이 단순하다.

```tsx title="DialogClick.tsx"
const DialogClick = ({ close }: { close: () => void }) => {
  const handleBackdropClick: React.MouseEventHandler<HTMLDivElement> = (event) => {
    if (event.target !== event.currentTarget) { // [!code highlight]
      return;
    }

    close();
  };

  const handleSaveClick: React.MouseEventHandler<HTMLButtonElement> = (event) => {
    event.preventDefault(); // [!code highlight]
    event.stopPropagation(); // [!code highlight]
    saveDraft();
  };

  return (
    <div className="backdrop" onClick={handleBackdropClick}>
      <section role="dialog" aria-modal="true">
        <form action="/posts">
          <button type="submit" onClick={handleSaveClick}>
            저장
          </button>
        </form>
      </section>
    </div>
  );
};
```

## 실수 포인트

- `target`을 handler가 붙은 container로 착각하지 않는다.
- Backdrop 닫기는 먼저 `target === currentTarget`으로 풀 수 있는지 본다.
- `stopPropagation`은 충돌이 나는 안쪽 action에만 둔다.
- Form submit이나 link 이동을 막으려면 `preventDefault`를 쓴다.
