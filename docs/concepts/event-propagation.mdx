# 이벤트 버블링 / 캡처링 / stopPropagation

## API reference

React 이벤트 handler는 브라우저 이벤트를 `SyntheticEvent`로 받는다. Handler 반환값은 React가 쓰지 않으므로, 전파와 기본 동작은 event 메서드로 제어한다.

| API | 선언 / 인자 | 동작 / 반환 |
|---|---|---|
| `React.SyntheticEvent<T = Element, E = Event>` | React가 넘기는 이벤트 객체 | DOM event를 감싼 객체다. `currentTarget`, `target`, `stopPropagation`, `preventDefault`, `nativeEvent`를 제공한다. |
| `onClick` / `onClickCapture` | `(event: React.MouseEvent<T>) => void` | 클릭 bubble / capture handler다. 반환값은 무시된다. |
| `onPointerDown` / `onPointerDownCapture` | `(event: React.PointerEvent<T>) => void` | 포인터가 눌리는 순간의 bubble / capture handler다. 마우스, 터치, 펜 입력을 같은 흐름으로 다룬다. |
| `event.currentTarget` | Handler가 붙은 element | React 타입에서 `T`로 좁혀진다. 부모 handler에서는 부모 element다. |
| `event.target` | 이벤트가 처음 시작된 DOM target | 자식 element일 수 있다. TypeScript에서는 보통 `EventTarget`으로 다룬다. |
| `event.stopPropagation()` | `() => void` | 남은 전파 단계를 막는다. 이미 실행된 handler는 되돌리지 않는다. |
| `event.preventDefault()` | `() => void` | 링크 이동, form submit 같은 기본 브라우저 동작을 막는다. 전파는 막지 않는다. |
| `event.nativeEvent` | 원본 DOM event | React event에 없는 DOM API가 필요할 때만 읽는다. 보통 handler 로직은 `SyntheticEvent`로 충분하다. |

```tsx title="handler-types.tsx"
const handleSaveClick: React.MouseEventHandler<HTMLButtonElement> = (event) => {
  console.log(event.currentTarget.name); // HTMLButtonElement
  event.stopPropagation();
};

const handleSurfacePointerDown: React.PointerEventHandler<HTMLDivElement> = (event) => {
  console.log(event.pointerType);
};
```

```tsx title="capture-bubble.tsx"
const CaptureBubble = () => (
  <section
    onClickCapture={() => {
      console.log("click capture");
    }}
    onClick={() => {
      console.log("click bubble");
    }}
    onPointerDownCapture={() => {
      console.log("pointer capture");
    }}
    onPointerDown={() => {
      console.log("pointer bubble");
    }}
  >
    <a
      href="/settings"
      onClick={(event) => {
        event.preventDefault();
        console.log(event.currentTarget.href);
      }}
    >
      설정
    </a>
  </section>
);
```

## 언제 쓰나

부모와 자식이 같은 클릭, 포인터, 키보드 이벤트를 함께 다룰 때 쓴다.

Modal backdrop, dropdown item, nested button처럼 안쪽 동작과 바깥 닫기 동작이 충돌하면 전파 흐름을 명시한다.

## 바로 쓰는 코드

React handler는 보통 target에서 시작해 부모로 올라간다. capture handler는 그 전에 바깥에서 안쪽으로 먼저 실행된다.

```tsx title="EventFlowPanel.tsx"
const EventFlowPanel = () => {
  const log = (message) => {
    console.log(message);
  };

  return (
    <section
      onClickCapture={() => {
        // Capture runs before the target button handler.
        log("section capture"); // [!code highlight]
      }}
      onClick={() => {
        log("section bubble");
      }}
    >
      <button
        type="button"
        onClick={(event) => {
          log("button target");
          // Stop parent React onClick handlers for this event.
          event.stopPropagation(); // [!code highlight]
        }}
      >
        저장
      </button>
    </section>
  );
};
```

Backdrop은 `target`과 `currentTarget` 비교가 먼저다. dialog 안쪽 클릭을 전부 막으려고 `stopPropagation`부터 넣으면 상위 로깅이나 단축키 흐름도 같이 끊긴다.

```tsx title="DialogClick.tsx"
const DialogClick = ({ close }) => {
  const handleBackdropClick = (event) => {
    if (event.target !== event.currentTarget) {
      return; // [!code highlight]
    }

    close();
  };

  const handleMenuButtonClick = (event) => {
    // Keep only this nested action from closing the parent menu.
    event.stopPropagation(); // [!code highlight]
    openNestedMenu();
  };

  return (
    <div className="backdrop" onClick={handleBackdropClick}>
      <section role="dialog" aria-modal="true">
        <button type="button" onClick={handleMenuButtonClick}>
          더 보기
        </button>
        {/* ... */}
      </section>
    </div>
  );
};
```

:::warning
`stopPropagation`은 현재 이벤트의 상위 handler를 막는다. analytics, outside click, 전역 shortcut처럼 상위에서 듣는 동작까지 필요한 화면에서는 먼저 구조를 확인한다.
:::

## 실수 포인트

- capture는 감시와 기록에 쓰고, 실제 UI 동작은 target 또는 bubble phase에 둔다.
- backdrop 닫기는 `target === currentTarget` 비교만으로 충분한 경우가 많다.
- nested action에서만 좁게 `stopPropagation`을 건다.
- `preventDefault`는 기본 브라우저 동작을 막는 API라 전파 차단과 역할이 다르다.
