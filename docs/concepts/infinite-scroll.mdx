# Infinite Scroll

## API reference

- `IntersectionObserver`: sentinel이 화면 기준 영역에 들어왔는지 관찰한다.
- `entry.isIntersecting`: sentinel이 기준 영역과 겹칠 때만 다음 page 요청을 시작한다.
- `rootMargin`: sentinel이 실제 끝에 닿기 전 미리 요청할 여백이다.
- `disconnect()`: 컴포넌트가 사라질 때 observer가 잡고 있던 관찰 대상을 모두 끊는다.
- `sentinel`: 목록 마지막에 두는 빈 DOM element다. 항목마다 관찰하지 않고 이 element 하나만 관찰한다.

## 쓰는 곳

- 피드, 검색 결과, 알림처럼 사용자가 아래로 계속 내려가며 보는 목록에 쓴다.
- 첫 화면에 모든 항목을 불러오면 응답이나 렌더링이 무거울 때 쓴다.
- footer나 하단 CTA를 반드시 보여야 하는 화면에는 pagination이나 더 보기 버튼이 더 낫다.

## 바로 쓰는 코드

sentinel 하나를 관찰하고, 이미 불러오는 중이거나 다음 page가 없으면 요청하지 않는다.

```tsx title="infinite-scroll-sentinel.tsx"
const sentinelRef = useRef<HTMLDivElement>(null);
const isLoadingRef = useRef(false);
const hasMoreRef = useRef(true);

const loadNextPage = async () => {
  if (isLoadingRef.current || !hasMoreRef.current) return;
  isLoadingRef.current = true;

  try {
    const page = await fetchNextPage();
    appendItems(page.items);
    hasMoreRef.current = page.hasMore;
  } finally {
    isLoadingRef.current = false;
  }
};

useEffect(() => {
  const sentinel = sentinelRef.current;
  if (!sentinel) return;
  const observer = new IntersectionObserver(
    ([entry]) => {
      if (!entry?.isIntersecting) return;
      void loadNextPage();
    },
    { rootMargin: "240px 0px" },
  );
  observer.observe(sentinel);

  return () => observer.disconnect();
}, []);
```

## 실수 포인트

- `isLoading` guard가 없으면 같은 sentinel 노출에서 같은 page를 여러 번 요청할 수 있다.
- `hasMore` guard가 없으면 마지막 page 뒤에도 계속 요청한다.
- 여러 항목을 각각 observe하면 observer callback과 cleanup이 불필요하게 늘어난다.
- `disconnect()`를 cleanup에서 빼면 unmount 뒤에도 observer가 남을 수 있다.
