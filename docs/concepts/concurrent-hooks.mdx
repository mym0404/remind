# React 최신 Hook / Concurrent UI

## API reference

- `useTransition()`: `[isPending, startTransition]`을 반환한다. transition으로 표시한 update는 다른 urgent update에 interrupt될 수 있다.
- `startTransition(action)`: `action`을 즉시 실행하고, 그 안에서 동기적으로 예약한 state update를 transition으로 표시한다.
- `isPending`: 같은 `useTransition`에서 시작한 transition이 아직 화면에 반영되지 않았는지 나타내는 boolean이다.
- `useDeferredValue(value, initialValue?)`: `value`의 소비 쪽 반영을 늦춘다. 첫 렌더에서는 `initialValue`가 있으면 그 값을 쓰고, 이후에는 이전 값을 먼저 유지한 뒤 background render가 끝나면 새 값으로 바뀐다.

## 쓰는 곳

입력, 클릭, 포커스처럼 바로 반응해야 하는 update와 목록, 차트, 탭 패널처럼 늦게 바뀌어도 되는 렌더링을 분리할 때 쓴다.

`useTransition`은 state update의 우선순위를 낮춘다. `useDeferredValue`는 값 자체를 복사하거나 debounce하지 않고, 그 값을 읽는 무거운 UI의 렌더링만 늦춘다.

## 바로 쓰는 코드

```tsx title="SearchResults.tsx"
const SearchResults = ({ items }: { items: string[] }) => {
  const [query, setQuery] = useState("");
  const [selectedItem, setSelectedItem] = useState<string>();
  const [isPending, startTransition] = useTransition();
  const deferredQuery = useDeferredValue(query);

  const visibleItems = useMemo(() => {
    const keyword = deferredQuery.toLowerCase();

    return items.filter((item) => item.toLowerCase().includes(keyword));
  }, [deferredQuery, items]);

  const selectItem = (item: string) => {
    startTransition(() => {
      setSelectedItem(item);
    });
  };

  return (
    <>
      <input value={query} onChange={(event) => setQuery(event.target.value)} />
      <p aria-live="polite">{isPending ? "선택 반영 중" : `${visibleItems.length}개`}</p>
      <ul aria-busy={query !== deferredQuery || isPending}>
        {visibleItems.map((item) => (
          <li key={item}>
            <button type="button" onClick={() => selectItem(item)}>
              {item === selectedItem ? `${item} 선택됨` : item}
            </button>
          </li>
        ))}
      </ul>
    </>
  );
};
```

## 실수 포인트

- controlled input의 `value`를 바꾸는 state update는 `startTransition`에 넣지 않는다.
- `useDeferredValue`는 고정 지연 시간, debounce, 네트워크 요청 제어가 아니다.
- deferred 값으로 계산한 목록과 원본 값으로 계산한 목록을 한 화면에서 섞지 않는다.
- `await` 뒤에 예약하는 state update까지 transition으로 다루려면 다시 `startTransition`으로 감싼다.
