# 성능 최적화 기초

## API reference

- `React.memo`: 이전 props와 다음 props가 같으면 component render를 건너뛸 수 있다. props가 매번 새 객체나 새 함수면 효과가 작다.
- `useMemo`: render 중 계산한 값을 dependency가 바뀔 때까지 다시 쓴다. 계산 비용이 크거나 memoized child에 같은 참조를 넘겨야 할 때만 둔다.
- `useCallback`: 함수 참조를 dependency가 바뀔 때까지 다시 쓴다. callback 내부에서 읽는 값은 dependency에 포함한다.
- `Profiler`: 감싼 subtree의 commit 비용을 측정한다. `actualDuration`으로 이번 render 비용을 보고, 병목 위치를 좁힌 뒤 최적화한다.
- dependency array: memoized value나 callback이 읽는 props, state, render 내부 값을 적는 입력 목록이다. 빠뜨리면 오래된 값을 읽고, 불필요하게 넣으면 캐시가 자주 무효화된다.

## 쓰는 곳

- Profiler에서 특정 subtree의 render 비용이 반복해서 크다고 보일 때 쓴다.
- `React.memo`로 감싼 child에 안정된 object, array, function props를 넘겨야 할 때 쓴다.
- 같은 render에서 바로 계산해도 싼 값은 그냥 계산한다.
- dependency를 맞추기 어렵다면 memoization보다 state 위치와 props 흐름을 먼저 줄인다.

## 바로 쓰는 코드

Profiler로 먼저 비용을 보고, 그 다음 같은 경계를 지나는 값과 callback만 고정한다.

```tsx title="measured-memo-boundary.tsx"
const onRender: ProfilerOnRenderCallback = (id, phase, actualDuration) => {
  console.table({ id, phase, actualDuration });
};

const SearchPanel = ({ query, items }: { query: string; items: Item[] }) => {
  const visibleItems = useMemo(
    () => items.filter((item) => item.name.includes(query)),
    [items, query],
  );

  const selectItem = useCallback((id: string) => {
    trackSelection(id);
  }, []);

  return (
    <Profiler id="SearchResults" onRender={onRender}>
      <SearchResults items={visibleItems} onSelect={selectItem} />
    </Profiler>
  );
};

const SearchResults = React.memo(
  ({ items, onSelect }: { items: Item[]; onSelect: (id: string) => void }) => (
    <ul>
      {items.map((item) => (
        <li key={item.id}>
          <button type="button" onClick={() => onSelect(item.id)}>
            {item.name}
          </button>
        </li>
      ))}
    </ul>
  ),
);
```

## 실수 포인트

- 측정 없이 `React.memo`, `useMemo`, `useCallback`을 먼저 깔지 않는다.
- dependency를 줄이려고 callback 내부에서 읽는 값을 빼지 않는다.
- 매번 새 object나 array를 props로 만들면 `React.memo` 경계가 쉽게 깨진다.
- `useMemo` 안에서 side effect를 실행하지 않는다.
- 싼 계산까지 캐시하면 비교와 캐시 유지 비용이 더 클 수 있다.
