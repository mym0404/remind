# MutationObserver

## API reference

`MutationObserver`는 React가 직접 관리하지 않는 DOM 변경을 비동기로 모아 알려준다. React state나 props로 이미 알 수 있는 변화라면 observer보다 그 값을 먼저 쓴다.

- `new MutationObserver(callback)`: observer를 만든다. `callback(records, observer)`는 DOM 변경이 모인 뒤 microtask에서 실행된다.
- `MutationRecord`: 변경 하나의 정보다. 주로 `type`, `target`, `addedNodes`, `removedNodes`, `attributeName`, `oldValue`를 확인한다.
- `observe(target, options)`: 감시할 `Node`와 변경 범위를 정한다. `childList`, `attributes`, `characterData` 중 하나는 반드시 켜야 한다.
- `takeRecords()`: 아직 callback으로 전달되지 않은 `MutationRecord[]`를 즉시 꺼내고 내부 queue를 비운다.
- `disconnect()`: 감시를 멈춘다. React effect cleanup에서 호출해 unmount 뒤 callback이 남지 않게 한다.

`observe` options는 필요한 것만 켠다.

- `childList`: 자식 node 추가와 제거를 본다.
- `attributes`: attribute 변경을 본다.
- `attributeFilter`: 지정한 attribute만 본다.
- `characterData`: text node 내용 변경을 본다.
- `subtree`: target 아래 하위 node까지 본다.

## 쓰는 곳

- 외부 script, browser extension, third-party widget이 DOM을 직접 바꿀 때 쓴다.
- portal이나 library가 붙인 DOM의 자식 node, `data-*`, `aria-*`, `class` 변화를 감지할 때 쓴다.
- React render 결과를 다시 관찰하는 용도로는 쓰지 않는다.

## 바로 쓰는 코드

```tsx title="useExternalDomChange.tsx"
const useExternalDomChange = (
  ref: React.RefObject<HTMLElement | null>,
  onChange: () => void,
) => {
  useEffect(() => {
    const target = ref.current;
    if (!target) {
      return;
    }

    const observer = new MutationObserver((records) => {
      const changed = records.some(
        ({ type, attributeName }) =>
          type === "childList" ||
          (type === "attributes" && attributeName === "data-state"),
      );

      if (changed) {
        onChange();
      }
    });
    observer.observe(target, {
      childList: true,
      attributes: true,
      attributeFilter: ["data-state"],
      subtree: true,
    });
    return () => {
      observer.takeRecords();
      observer.disconnect();
    };
  }, [onChange, ref]);
};
```

## 실수 포인트

- `subtree: true`를 `document.body` 같은 넓은 node에 걸면 작은 변경에도 callback이 자주 돈다.
- `attributes: true`는 `style`, `class` 변경도 볼 수 있으므로 callback에서 같은 state를 반복 갱신하지 않는다.
- `childList`만 켜면 attribute나 text 변경은 감지하지 못한다.
- cleanup 없이 observer를 남기면 unmount 뒤에도 callback이 실행될 수 있다.
