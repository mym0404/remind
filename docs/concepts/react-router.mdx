# React Router

## API reference

- `BrowserRouter`: 브라우저 주소와 history를 React Router에 연결한다.
- `Routes`/`Route`: 현재 URL에 맞는 route element를 고른다.
- `Link`/`NavLink`: 새로고침 없이 이동한다. 현재 route 스타일이 필요하면 `NavLink`를 쓴다.
- `useParams`: `:projectId` 같은 dynamic segment 값을 읽는다.
- `Outlet`: 부모 route 안에서 자식 route element가 렌더링될 자리다.
- `path="*"`: 앞 route가 매칭되지 않을 때 보여줄 fallback route다.

## 쓰는 곳

- 목록 URL과 상세 URL을 나눌 때 쓴다.
- 공통 layout 안에서 자식 화면만 바꿀 때 쓴다.
- 없는 주소를 404 화면으로 처리할 때 쓴다.

## 바로 쓰는 코드

```tsx title="App.tsx"
import { BrowserRouter, Link, Outlet, Route, Routes, useParams } from "react-router";

const projects = [{ id: "router", title: "라우팅" }];

export const App = () => (
  <BrowserRouter>
    <Routes>
      <Route path="/projects" element={<ProjectsLayout />}>
        <Route index element={<ProjectList />} />
        <Route path=":projectId" element={<ProjectDetail />} />
      </Route>
      <Route path="*" element={<NotFound />} />
    </Routes>
  </BrowserRouter>
);

const ProjectsLayout = () => <main><Outlet /></main>;

const ProjectList = () => (
  <ul>
    {projects.map((project) => (
      <li key={project.id}><Link to={`/projects/${project.id}`}>{project.title}</Link></li>
    ))}
  </ul>
);

const ProjectDetail = () => {
  const { projectId } = useParams<"projectId">();
  const project = projects.find((project) => project.id === projectId);
  return project ? <h1>{project.title}</h1> : <NotFound />;
};

const NotFound = () => <h1>페이지를 찾을 수 없습니다.</h1>;
```

## 실수 포인트

- route의 `:projectId` 이름과 `useParams`에서 읽는 이름을 다르게 쓰지 않는다.
- 부모 route에 자식 화면을 보여줄 `Outlet`을 빼먹지 않는다.
- `path="*"` route는 구체적인 route보다 뒤에 둔다.
- param 값은 문자열이거나 없을 수 있으므로 상세 데이터 조회 실패를 처리한다.
