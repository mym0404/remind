# 비동기 데이터 로딩

## API reference

- `fetch(input, init?)`: `Promise<Response>`를 반환한다. 요청 취소가 필요하면 `init.signal`에 `AbortSignal`을 넘긴다.
- `response.ok`: HTTP status가 200~299이면 `true`다. 404, 500 같은 HTTP error는 `fetch`가 자동으로 throw하지 않는다.
- `response.status`: 실패 메시지나 로그에 남길 숫자 상태 코드다.
- `response.json()`: response body를 한 번 읽어 JSON으로 파싱한다. 같은 body를 다시 읽을 수 없다.
- `AbortController`: `controller.signal`을 요청에 연결하고, effect cleanup에서 `controller.abort()`를 호출한다.
- `LoadState`: `loading`, `success`, `empty`, `error`를 한 state로 묶어 화면 분기를 명확히 한다.

## 쓰는 곳

- 화면이 열릴 때 목록이나 상세 데이터를 불러올 때 쓴다.
- 검색어, 필터, 페이지, route param이 바뀔 때 다시 요청할 때 쓴다.
- 이전 요청이 늦게 끝나도 최신 화면을 덮으면 안 될 때 쓴다.
- 서버 상태 라이브러리 없이 작은 범위의 요청 상태만 직접 제어할 때 쓴다.

## 바로 쓰는 코드

effect callback은 `async`가 될 수 없으므로 안쪽에서 async 함수를 만들고 `void`로 호출한다. 요청 시작, 성공, 빈 결과, 실패를 같은 state 흐름에서 갱신한다.

```tsx title="useUsers.tsx"
type LoadState =
  | { status: "loading" } // [!code highlight]
  | { status: "success"; users: { id: string; name: string }[] }
  | { status: "empty"; users: [] }
  | { status: "error"; message: string };

const useUsers = (query: string) => {
  const [state, setState] = useState<LoadState>({ status: "loading" });
  useEffect(() => {
    const controller = new AbortController(); // [!code highlight]
    const loadUsers = async () => {
      setState({ status: "loading" });
      try {
        const response = await fetch(`/api/users?q=${encodeURIComponent(query.trim())}`, {
          signal: controller.signal, // [!code highlight]
        });
        if (!response.ok) throw new Error(`요청 실패: ${response.status}`);
        const body: { users: { id: string; name: string }[] } = await response.json();
        if (controller.signal.aborted) return; // [!code highlight]
        setState(
          body.users.length > 0
            ? { status: "success", users: body.users }
            : { status: "empty", users: [] },
        );
      } catch (error) {
        if (error instanceof DOMException && error.name === "AbortError") return;
        setState({ status: "error", message: "사용자 목록을 불러오지 못했습니다." });
      }
    };

    void loadUsers();
    return () => controller.abort(); // [!code highlight]
  }, [query]);
  return state;
};
```

## 실수 포인트

- `useEffect(async () => {})`처럼 effect가 Promise를 반환하게 만들지 않는다.
- `response.ok`를 확인하지 않으면 HTTP error가 성공 흐름처럼 처리된다.
- cleanup에서 abort하지 않으면 unmount 뒤 state 갱신이나 낡은 응답 반영이 생길 수 있다.
- 빈 배열을 `success`로만 처리하면 사용자에게 로딩 종료와 빈 결과를 구분해 보여 주기 어렵다.
- abort된 요청은 `AbortError`로 들어오므로 사용자용 error UI로 바꾸지 않는다.
