# useReducer

## API reference

`useReducer`는 상태 변경 규칙을 `reducer`에 모으고, component에서는 `dispatch`로 action만 보낸다.

- `reducer(state, action)`: 현재 state와 action을 받아 다음 state를 반환한다.
- `dispatch(action)`: action을 reducer로 보내며, 다음 state를 직접 반환하지 않는다.
- `initialArg`/`init`: `init`이 없으면 `initialArg`가 초기 state이고, 있으면 첫 렌더링에서 `init(initialArg)`가 초기 state가 된다.
- action union: `type`별 payload를 union으로 나누면 `action.type`에 따라 필요한 값만 좁혀진다.
- immutable update: 기존 object와 array를 바꾸지 말고 새 값을 반환한다.

## 쓰는 곳

- 한 사용자 행동이 여러 state 조각을 함께 바꿀 때 쓴다.
- add, remove, reset처럼 상태 전이가 action 이름으로 설명될 때 쓴다.
- 입력값 하나나 boolean 하나처럼 단순한 상태는 `useState`가 더 읽기 쉽다.

## 바로 쓰는 코드

```tsx
import { useReducer } from "react";

type State = { count: number; history: number[] };

type Action =
  | { type: "increment"; amount: number }
  | { type: "reset" };

const reducer = (state: State, action: Action): State => {
  switch (action.type) {
    case "increment": {
      const count = state.count + action.amount;

      return {
        count,
        history: [...state.history, count],
      };
    }
    case "reset":
      return { count: 0, history: [] };
  }
};

const Counter = () => {
  const [state, dispatch] = useReducer(reducer, { count: 0, history: [] });

  return (
    <button
      type="button"
      onClick={() => dispatch({ type: "increment", amount: 1 })}
    >
      {state.count}
    </button>
  );
};
```

## 실수 포인트

- reducer 안에서 API 요청, 저장, analytics 전송을 하지 않는다.
- 기존 state에 `push`, `splice`, 직접 대입을 쓰지 않는다.
- action payload를 optional 속성 모음으로 만들지 않는다.
- 처리하지 않은 action을 `default`에서 조용히 숨기지 않는다.
