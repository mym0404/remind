# URL Search Params

## API reference

`URLSearchParams`는 query string을 읽고 고치는 브라우저 API다.

- `new URLSearchParams(init)`: `"?q=react"` 같은 문자열이나 기존 params를 읽는다.
- `get(name)`: 첫 값을 읽고 없으면 `null`을 반환한다.
- `set(name, value)`: 값을 문자열로 저장하고 기존 값을 덮어쓴다.
- `delete(name)`: key를 query string에서 제거한다.
- `toString()`: 앞의 `?` 없이 query string을 만든다.
- `useSearchParams()`: React Router에서 URL search와 setter를 받는다. setter는 navigation이므로 입력 동기화에는 `{ replace: true }`를 같이 쓴다.

## 쓰는 곳

- 검색어, 필터, 정렬, page처럼 공유 링크에 남겨야 하는 화면 state에 쓴다.
- 새로고침, 뒤로 가기, URL 공유 뒤에도 같은 조건을 복원해야 할 때 쓴다.

## 바로 쓰는 코드

```tsx title="SearchFilters.tsx"
const [searchParams, setSearchParams] = useSearchParams();

const query = searchParams.get("q") ?? "";

const syncQuery = (nextQuery: string) => {
  const nextParams = new URLSearchParams(searchParams); // [!code highlight]
  const trimmedQuery = nextQuery.trim();

  if (trimmedQuery) {
    nextParams.set("q", trimmedQuery);
    nextParams.delete("page");
  } else {
    nextParams.delete("q");
    nextParams.delete("page");
  }

  setSearchParams(nextParams, { replace: true }); // [!code highlight]
};
```

## 실수 포인트

- URL 값은 항상 문자열이라는 점을 잊지 않는다.
- 값이 비면 빈 문자열로 남기지 말고 `delete`한다.
- 기존 `searchParams`를 직접 바꾸지 말고 `new URLSearchParams(searchParams)`로 복사한다.
- URL과 React state를 둘 다 source of truth로 두지 않는다.
