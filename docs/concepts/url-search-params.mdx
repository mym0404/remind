# URL Search Params

## 언제 쓰나

- 검색어, 필터, 정렬처럼 공유 가능한 목록 조건을 URL에 남길 때.
- 새로고침, 공유 링크, 뒤로 가기에서 같은 조건을 복원해야 할 때.

## 바로 쓰는 코드

`useSearchParams`로 query string을 읽고 쓴다. URL을 검색 조건의 source of truth로 둔다.

```tsx title="ArticleListPage.tsx"
const categories = ["all", "docs", "api"];

const articles = [
  // ...
];

const isCategory = (value): value is Category =>
  categories.some((category) => category === value);

const ArticleListPage = () => {
  const [searchParams, setSearchParams] = useSearchParams();

  // Query strings are external input; normalize before rendering.
  const query = searchParams.get("q") ?? ""; // [!code highlight]
  const categoryParam = searchParams.get("category") ?? "all";
  const category = isCategory(categoryParam) ? categoryParam : "all"; // [!code highlight]
  const bookmarkedOnly = searchParams.get("bookmarked") === "true"; // [!code highlight]

  const updateSearchParams = ({
    nextBookmarkedOnly = bookmarkedOnly,
    nextCategory = category,
    nextQuery = query,
  }) => {
    const nextParams = new URLSearchParams(searchParams); // [!code highlight]
    const trimmedQuery = nextQuery.trim();

    if (trimmedQuery) {
      nextParams.set("q", trimmedQuery);
    } else {
      nextParams.delete("q");
    }

    if (nextCategory === "all") {
      nextParams.delete("category");
    } else {
      nextParams.set("category", nextCategory);
    }

    if (nextBookmarkedOnly) {
      nextParams.set("bookmarked", "true");
    } else {
      nextParams.delete("bookmarked");
    }

    // Avoid one history entry per keystroke.
    setSearchParams(nextParams, { replace: true }); // [!code highlight]
  };

  const visibleArticles = useMemo(() => {
    const normalizedQuery = query.trim().toLowerCase();

    return articles.filter((article) => {
      const matchesQuery = article.title.toLowerCase().includes(normalizedQuery);
      const matchesCategory =
        category === "all" || article.category === category;
      const matchesBookmark = !bookmarkedOnly || article.bookmarked;

      return matchesQuery && matchesCategory && matchesBookmark;
    });
  }, [bookmarkedOnly, category, query]);

  const handleQueryChange = (event) => {
    updateSearchParams({ nextQuery: event.target.value });
  };

  const handleCategoryChange = (event) => {
    const nextCategory = event.target.value;

    if (isCategory(nextCategory)) {
      updateSearchParams({ nextCategory });
    }
  };

  return (
    <section>
      <input onChange={handleQueryChange} value={query} />

      <select onChange={handleCategoryChange} value={category}>
        {/* ... */}
      </select>

      <input
        checked={bookmarkedOnly}
        onChange={(event) =>
          updateSearchParams({ nextBookmarkedOnly: event.target.checked })
        }
        type="checkbox"
      />

      {visibleArticles.map((article) => (
        <article key={article.title}>{article.title}</article>
      ))}
    </section>
  );
};
```

## 실수 포인트

:::warning
URL과 React state를 둘 다 source of truth로 두면 새로고침, 공유 링크, 뒤로 가기에서 조건이 어긋난다.
:::

- enum성 필터는 허용값으로 좁힌 뒤 기본값으로 정규화한다.
- 기존 `searchParams`를 복사한 뒤 필요한 key만 `set` 또는 `delete`한다.
