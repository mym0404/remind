# ARIA / Keyboard Interaction

## 언제 쓰나

접근성은 native element로 먼저 해결한다. `button`, `input`, `select`, `dialog`처럼 의미와 키보드 동작을 이미 가진 요소를 우선한다.

ARIA는 이름, 상태, 관계만 보강한다. 키보드 동작은 직접 구현해야 한다.

이름은 `aria-labelledby`가 우선이다. 보이는 이름이 없을 때만 `aria-label`을 쓰고, 도움말이나 에러는 `aria-describedby`로 붙인다.

열고 닫히는 UI의 trigger에는 `aria-expanded`, `aria-controls`, `aria-haspopup`를 함께 둔다. 선택은 `aria-selected`, 켜짐과 꺼짐은 native `checked`나 `aria-checked`로 표현한다.

Tab은 컴포넌트 밖으로 이동하고, Arrow 키는 composite 안의 active item을 바꾼다. Enter와 Space는 실행이나 선택, Escape는 닫기다.

:::warning
`role`만 붙이면 의미만 바뀐다. focus 이동, 선택, 닫기, disabled 차단은 React handler와 DOM 상태로 직접 맞춰야 한다.
:::

## 바로 쓰는 코드

Roving `tabindex` 패턴이다. disabled item은 탐색되지만 선택되면 안 된다.

```tsx title="ActionPicker.tsx"
const actions = [
  { id: "share", label: "공유" },
  { id: "archive", label: "보관", disabled: true },
  { id: "delete", label: "삭제" },
];

const nextIndex = (currentIndex, step) =>
  (currentIndex + step + actions.length) % actions.length;

const ActionPicker = () => {
  const [isOpen, setIsOpen] = useState(false);
  const [activeIndex, setActiveIndex] = useState(0);
  const [selectedId, setSelectedId] = useState(actions[0].id);
  const triggerRef = useRef(null);
  const optionRefs = useRef([]);
  const titleId = useId();
  const listboxId = useId();

  useEffect(() => {
    if (isOpen) {
      // Keep DOM focus on the active option.
      optionRefs.current[activeIndex]?.focus(); // [!code highlight]
    }
  }, [activeIndex, isOpen]);

  const closePicker = () => {
    setIsOpen(false);
    requestAnimationFrame(() => triggerRef.current?.focus());
  };

  const selectAction = (action) => {
    if (action.disabled) {
      // aria-disabled does not block events.
      return; // [!code highlight]
    }

    setSelectedId(action.id);
    closePicker();
  };

  const handleOptionKeyDown = (
    event,
    index,
  ) => {
    if (event.key === "ArrowDown" || event.key === "ArrowUp") {
      event.preventDefault();
      setActiveIndex(nextIndex(index, event.key === "ArrowDown" ? 1 : -1)); // [!code highlight]
      return;
    }

    if (event.key === "Enter" || event.key === " ") {
      event.preventDefault();
      selectAction(actions[index]);
      return;
    }

    if (event.key === "Escape") {
      event.preventDefault();
      closePicker(); // [!code highlight]
    }
  };

  return (
    <section aria-labelledby={titleId}>
      <h2 id={titleId}>작업 선택</h2>

      <button
        ref={triggerRef}
        type="button"
        aria-expanded={isOpen}
        aria-controls={listboxId}
        aria-haspopup="listbox"
        onClick={() => setIsOpen((current) => !current)}
      >
        작업 열기
      </button>

      {isOpen ? (
        <div id={listboxId} role="listbox" aria-labelledby={titleId}>
          {actions.map((action, index) => {
            return (
              <div
                id={`${listboxId}-${action.id}`}
                key={action.id}
                ref={(element) => {
                  optionRefs.current[index] = element;
                }}
                className="picker-option"
                role="option"
                tabIndex={index === activeIndex ? 0 : -1}
                aria-selected={selectedId === action.id}
                aria-disabled={action.disabled}
                onFocus={() => setActiveIndex(index)}
                onClick={() => selectAction(action)}
                onKeyDown={(event) => handleOptionKeyDown(event, index)}
              >
                {action.label}
              </div>
            );
          })}
        </div>
      ) : undefined}
    </section>
  );
};
```

DOM focus를 항목마다 옮기기 어렵다면 `aria-activedescendant`를 쓴다. focus는 컨테이너에 남고 active item의 `id`만 바뀐다.

```tsx title="ActivedescendantListbox.tsx"
const ActivedescendantListbox = () => {
  const [activeIndex, setActiveIndex] = useState(0);
  const [selectedId, setSelectedId] = useState(actions[0].id);
  const listboxId = useId();
  const activeItem = actions[activeIndex];
  // This id must exist in the DOM.
  const activeDescendantId = `${listboxId}-${activeItem.id}`; // [!code highlight]

  const handleKeyDown = (event) => {
    if (event.key === "ArrowDown") {
      event.preventDefault();
      setActiveIndex((current) => nextIndex(current, 1));
      return;
    }

    if (event.key === "ArrowUp") {
      event.preventDefault();
      setActiveIndex((current) => nextIndex(current, -1));
      return;
    }

    if (event.key === "Enter" || event.key === " ") {
      event.preventDefault();

      if (!activeItem.disabled) {
        setSelectedId(activeItem.id);
      }
    }
  };

  return (
    <div
      id={listboxId}
      role="listbox"
      tabIndex={0}
      aria-label="작업"
      aria-activedescendant={activeDescendantId}
      onKeyDown={handleKeyDown}
    >
      {actions.map((action) => (
        <div
          id={`${listboxId}-${action.id}`}
          key={action.id}
          role="option"
          aria-selected={selectedId === action.id}
          aria-disabled={action.disabled}
        >
          {action.label}
        </div>
      ))}
    </div>
  );
};
```

```css title="focus.css"
.picker-option:focus-visible,
[role="listbox"]:focus-visible {
  outline: 2px solid currentColor;
  outline-offset: 2px;
}
```

## 실수 포인트

- 보이는 label을 `aria-label`로 덮지 않는다.
- 설명, 도움말, 에러는 `aria-describedby`로 연결한다.
- trigger의 `aria-expanded`, `aria-controls`, `aria-haspopup`는 실제 popup 상태와 맞춘다.
- option, tab은 `aria-selected`, custom checkbox, switch는 `aria-checked`를 쓴다.
- `aria-disabled="true"`는 클릭과 키 입력을 막지 않으므로 handler에서 직접 막아야 한다.
- 실제 `disabled`는 form control을 비활성화하고 Tab 순서에서도 제외한다. 탐색 가능한 disabled item이 필요하면 `aria-disabled`를 쓴다.
- Toast나 비동기 완료 메시지는 `aria-live`나 `role="status"`로 노출한다.
- Arrow 키 이동 중 Tab까지 가로채지 않는다.
- focus indicator는 지우지 않는다. 디자인을 바꾸더라도 `:focus-visible` 상태는 남긴다.
- `aria-activedescendant`를 쓸 때 참조하는 `id`가 실제 DOM에 없으면 active item을 전달할 수 없다.
