# ARIA / Keyboard Interaction

## API reference

키보드 동작은 `onKeyDown`에서 처리한다. handler는 값을 반환해도 기본 동작을 막지 못한다. 스크롤, Space 클릭, Enter 제출을 막아야 할 때만 `event.preventDefault()`를 호출한다.

| API | 핵심 계약 |
|---|---|
| `KeyboardEventHandler<T>` | `event.key`와 `event.currentTarget`을 읽는다. 반환값은 쓰지 않는다. |
| `event.key` | Arrow, Home, End, Enter, `" "`, Escape, Tab을 분기한다. Space 값은 `" "`다. |
| `event.preventDefault()` | Arrow 스크롤, Space 클릭, Enter 제출처럼 충돌하는 기본 동작만 막는다. Tab 이동은 보통 막지 않는다. |
| `tabIndex` | `0`은 Tab으로 진입, `-1`은 코드로만 focus 이동이다. roving focus에서 active item만 `0`을 둔다. |
| `focus()` | 닫을 때 trigger로 focus를 돌려준다. DOM focus를 옮긴 뒤 상태와 ARIA도 같은 값이어야 한다. |
| `aria-*` | 이름, 상태, 관계만 전달한다. 클릭 차단, 선택 변경, focus 이동은 직접 구현한다. |

컴포넌트의 `onKeyDown` prop만 쓰면 별도 cleanup은 없다. `document.addEventListener("keydown", handler)`처럼 전역 listener를 붙이면 `useEffect` cleanup에서 같은 handler를 제거한다.

## 쓰는 곳

- native `button`, `input`, `select`, `dialog`로 해결되지 않는 composite UI에 쓴다.
- combobox, listbox, tabs, menu처럼 focus와 선택 상태가 따로 움직이는 UI에 쓴다.
- popup trigger에는 `aria-expanded`, `aria-controls`, `aria-haspopup`를 실제 열린 상태와 맞춘다.
- 보이는 이름은 `aria-labelledby`로 연결하고, 보이는 이름이 없을 때만 `aria-label`을 쓴다.
- DOM focus를 항목마다 옮기기 어렵다면 container에 focus를 두고 `aria-activedescendant`로 active option의 실제 `id`를 가리킨다.

## 바로 쓰는 코드

Arrow는 active item을 바꾸고, Enter와 Space는 선택한다. disabled item은 `aria-disabled`만으로 막히지 않으므로 handler에서 한 번 더 막는다.

```ts title="listbox-keydown.ts"
const move = (index: number, step: 1 | -1, length: number) =>
  (index + step + length) % length;

const handleKeyDown = (event: React.KeyboardEvent<HTMLDivElement>) => {
  if (event.key === "ArrowDown" || event.key === "ArrowUp") {
    event.preventDefault(); // [!code highlight]
    setActiveIndex((index) =>
      move(index, event.key === "ArrowDown" ? 1 : -1, actions.length),
    );
    return;
  }

  if (event.key === "Enter" || event.key === " ") {
    event.preventDefault();
    const action = actions[activeIndex];
    if (action.disabled) return; // [!code highlight]
    selectAction(action.id);
  }

  if (event.key === "Escape") {
    closeListbox(); // [!code highlight]
  }
};
```

```tsx title="activedescendant-listbox.tsx"
<div
  role="listbox"
  tabIndex={0}
  aria-activedescendant={activeOptionId}
  onKeyDown={handleKeyDown}
>
  <div id={activeOptionId} role="option" aria-selected={isSelected}>
    {label}
  </div>
</div>
```

## 실수 포인트

- `role`만 붙이면 의미만 바뀐다.
- ARIA 상태를 바꿔도 focus 이동, 선택, 닫기, disabled 차단은 자동으로 생기지 않는다.
- 화면 state와 `aria-expanded`, `aria-selected`, `aria-disabled` 값을 따로 계산하지 않는다.
- `aria-disabled="true"`는 이벤트를 막지 않는다.
- 실제 `disabled`는 form control을 Tab 순서에서 제외한다.
- Arrow 이동 중 Tab까지 가로채지 않는다.
- `aria-activedescendant`가 가리키는 `id`는 실제 DOM에 있어야 한다.
- focus indicator는 지우지 않는다. 디자인을 바꾸더라도 `:focus-visible` 상태는 남긴다.
