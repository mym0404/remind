# Ref / Imperative API

## API reference

- `useRef(initialValue)`: render 사이에 같은 ref 객체를 유지한다. `current`를 바꿔도 rerender가 생기지 않는다.
- DOM ref: JSX `ref`에 연결하면 mount 뒤 `current`가 DOM node를 가리킨다. event handler나 effect에서 읽는다.
- mutable ref: timer id, observer, 이전 값처럼 화면에 직접 그리지 않는 값을 담는다.
- `useImperativeHandle(ref, createHandle, deps)`: parent가 받는 `ref.current`를 직접 정한다. DOM node 전체 대신 필요한 method만 노출한다.

## 쓰는 곳

- input focus, scroll 이동처럼 DOM method를 event에서 호출할 때 쓴다.
- timer id, observer instance, 이전 요청 id처럼 render와 무관한 값을 보관할 때 쓴다.
- parent가 child 내부 DOM을 몰라도 되는 작은 imperative API가 필요할 때 쓴다.

## 바로 쓰는 코드

```tsx title="SearchInput.tsx"
type SearchInputHandle = {
  focus: () => void;
  clear: () => void;
};

type SearchInputProps = {
  value: string;
  onChange: (value: string) => void;
};

const SearchInput = forwardRef<SearchInputHandle, SearchInputProps>(
  ({ value, onChange }, ref) => {
    const inputRef = useRef<HTMLInputElement>(null); // [!code highlight]

    useImperativeHandle(
      ref,
      () => ({
        focus: () => inputRef.current?.focus(),
        clear: () => {
          onChange("");
          inputRef.current?.focus();
        },
      }),
      [onChange], // [!code highlight]
    );

    return <input ref={inputRef} value={value} onChange={({ target }) => onChange(target.value)} />;
  },
);
```

## 실수 포인트

- `ref.current`를 render 결과를 정하는 값으로 쓰지 않는다.
- render 중 DOM ref를 읽지 않는다.
- parent에게 DOM node 전체를 넘기는 것을 기본값으로 삼지 않는다.
- `useImperativeHandle` callback이 props나 state를 읽으면 `deps`에 넣는다.
