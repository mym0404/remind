# Autocomplete

## API reference

autocomplete은 DOM focus를 input에 둔 채 추천 목록만 바꾼다. input은 `combobox`, 목록은 `listbox`, 항목은 `option`으로 노출한다.

| 대상 | role / 속성 | 기준 |
| --- | --- | --- |
| input | `role="combobox"` | 사용자가 타이핑하는 제어 지점이다. |
| input | `aria-expanded` | 목록이 열리면 `true`, 닫히면 `false`다. |
| input | `aria-controls` | 연결할 listbox의 `id`를 가리킨다. |
| input | `aria-activedescendant` | 방향키로 가리키는 option의 `id`를 넣는다. |
| list | `role="listbox"` | 추천 항목 묶음이다. |
| item | `role="option"` | 선택 가능한 항목이다. |

키보드는 `ArrowDown`, `ArrowUp`, `Enter`, `Escape`만 먼저 잡는다. 한글 조합 중에는 `event.nativeEvent.isComposing`이 `true`일 수 있으므로 `Enter` 선택을 실행하지 않는다.

## 쓰는 곳

짧은 입력으로 후보를 좁히고 키보드로 고를 때 쓴다. 화면 값은 즉시 바꾸고, 추천 조회는 debounce된 값으로 따로 보낸다.

## 바로 쓰는 코드

active option의 `id`만 input에 알려준다.

```tsx title="autocomplete-core.tsx"
const listboxId = useId();
const [isOpen, setIsOpen] = useState(false);
const [activeIndex, setActiveIndex] = useState<number>();
const activeOptionId =
  activeIndex === undefined ? undefined : `${listboxId}-${activeIndex}`;
<>
  <input
    role="combobox" // [!code highlight]
    aria-expanded={isOpen} // [!code highlight]
    aria-controls={listboxId} // [!code highlight]
    aria-activedescendant={activeOptionId} // [!code highlight]
    onKeyDown={(event) => {
      if (event.nativeEvent.isComposing) return; // [!code highlight]
      if (options.length === 0) return;
      if (event.key === "ArrowDown") {
        event.preventDefault();
        setIsOpen(true);
        setActiveIndex((index = -1) => (index + 1) % options.length);
      }
      if (event.key === "ArrowUp") {
        event.preventDefault();
        setActiveIndex((index = 0) => (index - 1 + options.length) % options.length);
      }
      if (event.key === "Enter" && activeIndex !== undefined) selectOption(options[activeIndex]);
      if (event.key === "Escape") setIsOpen(false);
    }}
  />
  {isOpen && (
    <ul id={listboxId} role="listbox">
      {options.map((option, index) => (
        <li id={`${listboxId}-${index}`} role="option" key={option.id}>{option.label}</li>
      ))}
    </ul>
  )}
</>;
```

## 실수 포인트

- 방향키 이동 때 DOM focus를 option으로 옮기지 않는다.
- input 값과 선택된 항목을 같은 state로만 처리하지 않는다.
- 선택 직후 같은 값으로 목록을 다시 열지 않는다.
- `loading`, `error`, `empty`를 같은 상태로 뭉개지 않는다.
- 바깥을 눌러 닫을 때 input 값은 지우지 않는다.
