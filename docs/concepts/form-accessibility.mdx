# Form 접근성

## API reference

폼 접근성은 필드 이름, 오류 설명, 그룹 이름, submit 실패 후 focus 위치를 DOM 관계로 고정하는 규칙이다.

- `label htmlFor={fieldId}` + `id={fieldId}`: 화면에 보이는 label을 control 이름으로 연결한다.
- `aria-invalid={hasError ? true : undefined}`: 오류가 있을 때만 invalid 상태를 노출한다.
- `aria-describedby={ids}`: 도움말과 오류 메시지의 현재 렌더링된 `id`만 공백으로 연결한다.
- `fieldset`/`legend`: radio, checkbox처럼 같은 질문에 속한 control group의 이름을 만든다.
- `focus()`: submit 실패 후 첫 invalid field로만 이동한다. 입력 중에는 focus를 빼앗지 않는다.

## 쓰는 곳

- validation error와 help text가 있는 input, textarea에 쓴다.
- radio/checkbox group처럼 control은 여러 개지만 질문은 하나일 때 쓴다.
- submit 실패 뒤 사용자가 바로 고칠 위치로 이동해야 할 때 쓴다.

## 바로 쓰는 코드

```tsx title="ContactForm.tsx"
const ContactForm = () => {
  const [error, setError] = useState<string>();
  const emailRef = useRef<HTMLInputElement>(null);
  const emailId = useId();
  const helpId = `${emailId}-help`;
  const errorId = `${emailId}-error`;

  const handleSubmit = (event: React.FormEvent<HTMLFormElement>) => {
    event.preventDefault();
    const nextError = validateEmail(new FormData(event.currentTarget));
    setError(nextError);

    if (nextError) emailRef.current?.focus(); // [!code highlight]
  };

  return (
    <form noValidate onSubmit={handleSubmit}>
      <label htmlFor={emailId}>이메일</label>
      <input
        id={emailId}
        ref={emailRef}
        type="email"
        aria-invalid={error ? true : undefined} // [!code highlight]
        aria-describedby={`${helpId}${error ? ` ${errorId}` : ""}`} // [!code highlight]
      />
      <p id={helpId}>답변을 받을 이메일을 입력하세요.</p>
      {error ? <p id={errorId}>{error}</p> : undefined}
    </form>
  );
};
```

`aria-describedby`는 DOM에 있는 `id`만 가리켜야 한다. 오류 메시지를 조건부 렌더링하면 `aria-invalid`와 오류 `id`도 같은 조건으로 계산한다.

## 실수 포인트

- placeholder를 label 대체로 쓰지 않는다.
- 보이지 않는 error `id`를 `aria-describedby`에 계속 연결하지 않는다.
- 입력 중 validation 때문에 focus를 이동하지 않는다.
- radio/checkbox 묶음을 group 이름 없이 만들지 않는다.
