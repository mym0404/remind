# Toast

## API reference

- queue: 화면에 떠 있는 toast 목록이다. 새 항목은 기존 항목을 덮어쓰지 않고 뒤에 붙인다.
- `aria-live`: 일반 알림은 `polite`로 알리고, 즉시 알려야 하는 실패만 `assertive`로 분리한다.
- auto dismiss timer: toast마다 제거 timer를 만든다.
- manual close: 닫기 버튼은 같은 제거 함수를 호출해 특정 `id`만 지운다.
- cleanup: 수동 닫기, 자동 제거, unmount에서 남은 timer를 정리한다.

## 쓰는 곳

Toast는 저장 완료, 복사 완료, 삭제 실패처럼 방금 한 작업의 결과를 짧게 알려줄 때 쓴다. 사용자의 선택이나 복구가 필요하면 toast 대신 dialog, inline error, undo action을 먼저 검토한다.

## 바로 쓰는 코드

Provider가 queue와 timer를 소유한다. 사용하는 쪽은 `useToast().showToast()`만 호출한다.

```tsx title="ToastProvider.tsx"
type ToastInput = { message: string; tone?: "info" | "success" | "error"; duration?: number };
type ToastItem = Required<ToastInput> & { id: string };
type ToastContextValue = {
  showToast: (input: ToastInput) => string;
  removeToast: (id: string) => void;
};

const ToastContext = createContext<ToastContextValue | undefined>(undefined);

export const ToastProvider = ({ children }: { children: React.ReactNode }) => {
  const [toasts, setToasts] = useState<ToastItem[]>([]);
  const timersRef = useRef<Record<string, number>>({});

  const removeToast = useCallback((id: string) => {
    window.clearTimeout(timersRef.current[id]);
    delete timersRef.current[id];
    setToasts((current) => current.filter((toast) => toast.id !== id));
  }, []);

  const showToast = useCallback(
    ({ message, tone = "info", duration = 3000 }: ToastInput) => {
      const id = crypto.randomUUID();
      setToasts((current) => [...current, { id, message, tone, duration }]); // [!code highlight]
      timersRef.current[id] = window.setTimeout(() => removeToast(id), duration);
      return id;
    },
    [removeToast],
  );

  useEffect(() => {
    return () => Object.values(timersRef.current).forEach(window.clearTimeout); // [!code highlight]
  }, []);

  const value = useMemo(() => ({ showToast, removeToast }), [showToast, removeToast]);

  return (
    <ToastContext.Provider value={value}>
      {children}
      <div role="status" aria-live="polite" aria-atomic="true">
        {toasts.map((toast) => (
          <section key={toast.id} data-tone={toast.tone}>
            {toast.message}
            <button type="button" onClick={() => removeToast(toast.id)}>닫기</button>
          </section>
        ))}
      </div>
    </ToastContext.Provider>
  );
};

export const useToast = () => {
  const context = useContext(ToastContext);
  if (context === undefined) throw new Error("useToast must be used within ToastProvider");
  return context;
};
```

## 실수 포인트

- Provider 밖에서 toast state를 흩어 놓으면 queue, timer, live region이 서로 어긋난다.
- `setToasts([...toasts, toast])`는 빠른 연속 호출에서 이전 queue를 놓칠 수 있다.
- 자동 제거와 수동 닫기가 다른 제거 경로를 쓰면 timer가 남기 쉽다.
- `aria-live="assertive"`를 모든 toast에 쓰면 screen reader 흐름을 과하게 끊는다.
- unmount cleanup이 없으면 provider가 사라진 뒤 timer callback이 실행될 수 있다.
