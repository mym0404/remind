# ResizeObserver

## API reference

`ResizeObserver`는 element의 content box 크기가 바뀐 뒤 callback을 실행한다. window 크기가 그대로여도 부모 layout, font, image, sidebar 상태 때문에 element 크기는 달라질 수 있다.

`new ResizeObserver(callback)`의 callback은 `(entries, observer)`를 받는다. `entries`에는 크기가 바뀐 element별 `ResizeObserverEntry`가 들어오고, `observer`는 callback을 실행한 instance다.

`ResizeObserverEntry`는 관찰 대상과 측정 결과를 담는다. 이 문서에서는 `contentRect`만 읽는다.

| 항목 | 설명 |
|---|---|
| `entry.target` | 크기가 바뀐 element |
| `entry.contentRect.width` | content box의 가로 크기 |
| `entry.contentRect.height` | content box의 세로 크기 |

관찰은 `observe(element)`로 시작한다. 특정 element만 빼려면 `unobserve(element)`를 쓰고, component cleanup에서는 `disconnect()`로 observer가 잡고 있는 관찰을 모두 끊는다.

## 쓰는 곳

- container 너비에 따라 layout mode를 바꿀 때
- popover나 floating panel의 기준 element 크기를 다시 읽어야 할 때
- text, image, sidebar 접힘처럼 window resize 없이 element 크기가 바뀔 때

## 바로 쓰는 코드

```tsx title="useElementSize.ts"
const useElementSize = () => {
  const ref = useRef<HTMLDivElement>(null);
  const [size, setSize] = useState({ height: 0, width: 0 });

  useEffect(() => {
    const element = ref.current;
    if (!element) {
      return;
    }

    const observer = new ResizeObserver(([entry]) => {
      const { height, width } = entry.contentRect; // [!code highlight]
      setSize((currentSize) => {
        if (currentSize.width === width && currentSize.height === height) {
          return currentSize;
        }
        return { height, width };
      });
    });

    observer.observe(element); // [!code highlight]
    return () => {
      observer.disconnect(); // [!code highlight]
    };
  }, []);

  return { ref, size };
};
```

## 실수 포인트

- `window.resize`는 element 자체의 크기 변화를 모두 잡지 못한다.
- `observe`는 ref가 실제 element를 가리킨 뒤 호출한다.
- 같은 `width`와 `height`면 같은 state 객체를 반환해 불필요한 render를 피한다.
- cleanup에서 `disconnect()`를 빠뜨리면 사라진 component의 element 관찰이 남을 수 있다.
- observer callback 안에서 측정 대상의 크기를 다시 바꾸는 DOM write를 바로 넣으면 loop 경고가 날 수 있다.
