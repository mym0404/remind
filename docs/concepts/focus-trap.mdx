# Focus Trap

## API reference

- `document.activeElement`: trap을 열기 전 focus된 요소를 읽는다. `HTMLElement`가 아니거나 `body`면 복귀 대상으로 쓰지 않는다.
- `focus()`: 첫 focusable 요소로 focus를 옮기고, Tab 순환 경계에서 반대쪽 요소로 다시 보낸다. 스크롤 튐을 줄이려면 `preventScroll`을 함께 쓴다.
- focusable selector: `querySelectorAll()`로 후보를 찾되, disabled, hidden, `aria-hidden`, 음수 `tabIndex`는 다시 걸러야 한다.
- Tab/Escape `keydown`: Tab은 순환시키고, Escape는 닫기 동작에 연결한다.
- cleanup: 닫힐 때 `keydown` listener를 제거하고, 열기 전 focus가 아직 연결되어 있으면 복귀시킨다.

## 쓰는 곳

Focus Trap은 modal이나 drawer처럼 열린 영역을 먼저 처리해야 하는 UI에 쓴다. 열린 동안 Tab focus가 영역 밖으로 빠지지 않게 하고, 닫힌 뒤에는 열기 전 위치로 되돌린다.

Focus Trap은 focus 이동만 맡는다. 배경 클릭 차단, scroll lock, portal, 닫힘 상태 관리는 dialog 쪽에서 따로 처리한다.

## 바로 쓰는 코드

```tsx title="activateFocusTrap.ts"
const focusableSelector = ["a[href]", "button", "input", "select", "textarea", "[tabindex]"].join(",");
const getActiveElement = () => {
  const element = document.activeElement;
  return element instanceof HTMLElement && element !== document.body ? element : undefined;
};

const getTabbableElements = (container: HTMLElement) =>
  [...container.querySelectorAll<HTMLElement>(focusableSelector)].filter(
    (element) => !element.matches(":disabled") && !element.closest("[hidden], [inert], [aria-hidden='true']") && element.tabIndex >= 0,
  );

const activateFocusTrap = (container: HTMLElement, onEscape: () => void) => {
  const previousFocus = getActiveElement();
  (getTabbableElements(container)[0] ?? container).focus({ preventScroll: true });
  const handleKeyDown = (event: KeyboardEvent) => {
    if (event.key === "Escape") return onEscape();
    if (event.key !== "Tab") return;

    const elements = getTabbableElements(container);
    const first = elements[0] ?? container;
    const last = elements[elements.length - 1] ?? container;
    const active = getActiveElement();
    const shouldWrap = !active || !container.contains(active) || (event.shiftKey ? active === first : active === last);

    if (!shouldWrap) return;
    event.preventDefault();
    (event.shiftKey ? last : first).focus({ preventScroll: true });
  };

  document.addEventListener("keydown", handleKeyDown);
  return () => {
    document.removeEventListener("keydown", handleKeyDown);
    if (previousFocus?.isConnected) previousFocus.focus({ preventScroll: true });
  };
};
```

## 실수 포인트

- focusable selector 결과를 그대로 믿으면 disabled나 숨김 요소로 focus가 이동할 수 있다.
- Tab을 누를 때마다 focusable 목록을 다시 계산하지 않으면 동적으로 바뀐 버튼이나 입력값을 놓친다.
- focus가 trap 밖에서 시작한 경우도 첫 요소나 마지막 요소로 다시 끌어와야 한다.
- 닫힐 때 listener를 제거하지 않으면 닫힌 UI의 focus 로직이 계속 실행된다.
- 열기 전 요소가 unmount될 수 있으면 focus 복귀 전에 연결 여부를 확인한다.
