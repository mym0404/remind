# Portal

## API reference

`createPortal(children, domNode, key?)`는 `children`을 React tree에 남겨 둔 채, 실제 DOM만 `domNode` 아래에 렌더링한다.

```tsx title="Declaration"
declare function createPortal(
  children: React.ReactNode,
  domNode: Element | DocumentFragment,
  key?: string | null,
): React.ReactPortal;
```

- `children`: Portal로 보낼 React node다.
- `domNode`: 이미 존재하는 DOM node다. 없으면 직접 만들고 mount 뒤에 연결해야 한다.
- `key`: 같은 target에 여러 Portal을 렌더링할 때 구분이 필요하면 넘긴다.

대상 DOM node를 컴포넌트가 만들었다면 unmount 때 같이 제거한다.

```tsx title="usePortalTarget.tsx"
const usePortalTarget = (targetId: string) => {
  const [target, setTarget] = useState<HTMLElement>();

  useEffect(() => {
    const existingTarget = document.getElementById(targetId);

    if (existingTarget) {
      setTarget(existingTarget);
      return;
    }

    const createdTarget = document.createElement("div");
    createdTarget.id = targetId;
    document.body.appendChild(createdTarget);
    setTarget(createdTarget);

    return () => {
      createdTarget.remove(); // [!code highlight]
    };
  }, [targetId]);

  return target;
};
```

Portal 내부 event는 DOM tree가 아니라 React tree 기준으로 전파된다. Modal, popover처럼 바깥 클릭을 다루는 UI에서는 내부 클릭이 부모 handler까지 올라가지 않게 막아야 한다.

```tsx title="Popover.tsx"
const Popover = ({ anchorId }: { anchorId: string }) => {
  const target = usePortalTarget("portal-root");

  if (target === undefined) {
    return null;
  }

  return createPortal(
    <div
      aria-labelledby={anchorId}
      className="popover"
      onClick={(event) => event.stopPropagation()} // [!code highlight]
      role="dialog"
    >
      빠른 작업
    </div>,
    target,
  );
};
```

## 언제 쓰나

Portal은 Modal, toast, dropdown처럼 떠 있는 UI를 부모 DOM 밖에 렌더링할 때 쓴다. React tree는 그대로라 Context, state, event 흐름은 유지된다.

Portal은 z-index 값을 키우는 해결책이 아니다. 레이어 UI를 한 위치로 모아 잘림과 stacking context 충돌을 줄이는 방법이다.

:::warning
Portal은 DOM 위치만 바꾼다. focus trap, Escape 닫기, 스크롤 잠금, 접근성 속성은 모달 컴포넌트에서 따로 처리해야 한다.
:::

## 바로 쓰는 코드

대상 node가 없으면 mount 후 만들고, 준비 전에는 렌더링하지 않는다.

```tsx title="Portal.tsx"
const PORTAL_ROOT_ID = "portal-root";

const usePortalTarget = (targetId: string) => {
  const [target, setTarget] = useState<HTMLElement>();

  useEffect(() => {
    const existingTarget = document.getElementById(targetId);

    if (existingTarget) {
      setTarget(existingTarget);
      return;
    }

    const createdTarget = document.createElement("div");
    createdTarget.id = targetId;
    document.body.appendChild(createdTarget); // [!code highlight]
    setTarget(createdTarget);

    return () => {
      createdTarget.remove(); // [!code highlight]
    };
  }, [targetId]);

  return target;
};

const Portal = ({
  children,
  targetId = PORTAL_ROOT_ID,
}: {
  children: React.ReactNode;
  targetId?: string;
}) => {
  const target = usePortalTarget(targetId);

  if (target === undefined) {
    // createPortal needs a real DOM node.
    return null; // [!code highlight]
  }

  return createPortal(children, target); // [!code highlight]
};
```

```tsx title="InviteModal.tsx"
const PortalModalExample = () => {
  const [isOpen, setIsOpen] = useState(false);

  const closeModal = () => {
    setIsOpen(false);
  };

  const handleBackdropClick = (event) => {
    event.stopPropagation();

    if (event.target === event.currentTarget) {
      closeModal(); // [!code highlight]
    }
  };

  const stopDialogClick = (event) => {
    // Portal events still bubble through the React tree.
    event.stopPropagation();
  };

  return (
    <>
      <button type="button" onClick={() => setIsOpen(true)}>
        초대하기
      </button>

      {isOpen ? (
        <Portal>
          <div className="modal-backdrop" onClick={handleBackdropClick}>
            <section
              aria-labelledby="invite-title"
              aria-modal="true"
              className="modal"
              onClick={stopDialogClick} // [!code highlight]
              role="dialog"
            >
              <h2 id="invite-title">팀원 초대</h2>
              {/* ... */}
              <button type="button" onClick={closeModal}>
                닫기
              </button>
            </section>
          </div>
        </Portal>
      ) : undefined}
    </>
  );
};
```

Dropdown은 floating element만 Portal로 보내고, trigger 위치 계산은 따로 처리한다.

## 실수 포인트

- Portal 내부 event는 DOM tree가 아니라 React tree 기준으로 bubble된다.
- Portal 안쪽 클릭이 상위 `onClick`을 건드리면 내부에서 `event.stopPropagation()`을 호출하거나 Portal을 React tree의 더 위로 옮긴다.
- z-index가 계속 꼬이면 Portal root를 여러 곳에 만들기보다 레이어를 모을 위치를 하나로 정한다.
