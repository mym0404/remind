# Portal

## API reference

- `createPortal(children, domNode)`: `children`을 React tree에 남긴 채 실제 DOM만 `domNode` 아래에 렌더링한다.
- target node: `domNode`는 `createPortal` 호출 전에 필요하다. 없으면 mount 이후 만들고, 준비 전에는 렌더링하지 않는다.
- React event bubbling: Portal 내부 event는 DOM parent가 아니라 React parent를 따라 bubble된다.
- cleanup: 컴포넌트가 직접 만든 target node만 unmount 때 제거한다.

## 쓰는 곳

Portal은 modal, toast, popover처럼 부모 DOM의 `overflow`나 stacking context에서 벗어나야 하는 UI에 쓴다.

React tree는 그대로라 Context와 state 흐름은 유지된다. DOM 위치만 바뀌므로 focus trap, Escape 닫기, 스크롤 잠금은 Portal이 아니라 레이어 컴포넌트가 처리한다.

## 바로 쓰는 코드

target이 없으면 mount 후 만들고, 직접 만든 target만 cleanup한다.

```tsx title="Portal.tsx"
const PORTAL_ROOT_ID = "portal-root";

const usePortalTarget = () => {
  const [target, setTarget] = useState<HTMLElement>();
  useEffect(() => {
    const existingTarget = document.getElementById(PORTAL_ROOT_ID);

    if (existingTarget) {
      setTarget(existingTarget);
      return;
    }

    const createdTarget = document.createElement("div");
    createdTarget.id = PORTAL_ROOT_ID;
    document.body.appendChild(createdTarget);
    setTarget(createdTarget);

    return () => {
      createdTarget.remove(); // [!code highlight]
    };
  }, []);

  return target;
};

const Portal = ({ children }: { children: React.ReactNode }) => {
  const target = usePortalTarget();
  return target ? createPortal(children, target) : null; // [!code highlight]
};
```

## 실수 포인트

- `domNode`가 없는 상태에서 `createPortal`을 호출하면 안 된다.
- 이미 있던 target node를 컴포넌트 cleanup에서 제거하면 다른 Portal까지 깨질 수 있다.
- Portal 안쪽 클릭도 React parent의 `onClick`으로 올라갈 수 있다.
- Portal은 z-index를 자동으로 해결하지 않는다. 레이어를 모을 target 위치를 먼저 정한다.
