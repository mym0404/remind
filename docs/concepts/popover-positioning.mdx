# Popover / Dropdown / Tooltip Positioning

## API reference

- `getBoundingClientRect()`: trigger와 popover의 현재 viewport 기준 rect를 읽는다.
- viewport boundary: `window.innerWidth`, `window.innerHeight`를 기준으로 화면 밖 overflow를 판단한다.
- flip: 기본 방향에 공간이 부족하면 반대 방향을 쓴다.
- shift: 선택한 방향은 유지하고, 좌우나 위아래로 넘친 좌표만 boundary 안으로 민다.
- scroll/resize cleanup: 열린 동안만 위치를 다시 계산하고, cleanup에서 같은 listener 옵션으로 제거한다.

## 쓰는 곳

Popover, Dropdown, Tooltip처럼 trigger 옆에 짧은 floating UI를 붙일 때 쓴다. `body` portal과 `position: fixed` 조합이면 `getBoundingClientRect()`의 viewport 좌표를 그대로 쓸 수 있어 가장 단순하다.

scroll container 안에 갇혀야 하는 UI라면 viewport 대신 container와 viewport의 교집합을 boundary로 잡는다. 그렇지 않으면 메뉴가 panel 밖으로 보이거나, 반대로 너무 일찍 잘릴 수 있다.

## 바로 쓰는 코드

```ts title="compute-popover-position.ts"
type Placement = "top" | "bottom";
type PopoverRect = Pick<DOMRectReadOnly, "width" | "height">;

const GAP = 8;

const clamp = (value: number, min: number, max: number) =>
  Math.min(Math.max(value, min), Math.max(min, max));

const computePopoverPosition = ({
  triggerRect,
  popoverRect,
  viewport,
}: {
  triggerRect: DOMRectReadOnly;
  popoverRect: PopoverRect;
  viewport: { width: number; height: number };
}) => {
  const bottomTop = triggerRect.bottom + GAP;
  const topTop = triggerRect.top - popoverRect.height - GAP;
  const canUseBottom = bottomTop + popoverRect.height <= viewport.height;
  const canUseTop = topTop >= 0;
  const placement: Placement = canUseBottom || !canUseTop ? "bottom" : "top";
  const rawTop = placement === "bottom" ? bottomTop : topTop;
  const rawLeft = triggerRect.left + triggerRect.width / 2 - popoverRect.width / 2;

  return {
    top: clamp(rawTop, 0, viewport.height - popoverRect.height),
    left: clamp(rawLeft, 0, viewport.width - popoverRect.width),
    placement,
  };
};
```

## 실수 포인트

- `getBoundingClientRect()` 값에 `scrollY`를 더하면 `position: fixed` popover가 스크롤 후 밀린다.
- flip만 하고 shift를 빼면 화면 끝 trigger에서 popover 일부가 잘린다.
- scroll listener를 bubble 단계에만 붙이면 내부 scroll container 움직임을 놓칠 수 있다.
- `resize`와 `scroll` cleanup에서 등록할 때와 다른 옵션을 쓰면 listener가 남는다.
