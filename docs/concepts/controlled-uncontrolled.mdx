# Controlled / Uncontrolled 조합

## API reference

- `value`와 `onChange`를 함께 주면 React state가 현재 값을 가진다.
- `defaultValue`는 DOM의 초기값만 정하고, 제출 시 `ref`나 `FormData`로 읽는다.
- checkbox의 선택 상태는 `checked`로 제어하고, 변경값은 `event.currentTarget.checked`에서 읽는다.
- 한 input은 mount 이후 controlled와 uncontrolled 사이를 오가면 안 된다.

## 쓰는 곳

- 입력 즉시 validation, preview, 다른 field 갱신이 필요하면 controlled를 쓴다.
- 제출 시점에 한 번만 값이 필요하면 uncontrolled가 더 단순하다.
- 재사용 컴포넌트는 호출자가 값의 주인이어야 할 때만 controlled API를 노출한다.

## 바로 쓰는 코드

```tsx title="ControlledText.tsx"
const ControlledText = () => {
  const [name, setName] = useState("");

  return (
    <label>
      이름
      <input
        value={name}
        onChange={(event) => setName(event.currentTarget.value)}
      />
    </label>
  );
};
```

```tsx title="UncontrolledForm.tsx"
const UncontrolledForm = () => {
  const submit: React.FormEventHandler<HTMLFormElement> = (event) => {
    event.preventDefault();
    const formData = new FormData(event.currentTarget);
    console.log(String(formData.get("name") ?? ""));
  };

  return (
    <form onSubmit={submit}>
      <input name="name" defaultValue="민지" />
      <button type="submit">저장</button>
    </form>
  );
};
```

## 실수 포인트

- `value`만 주고 `onChange`를 빼면 입력이 잠긴다.
- 값이 없을 수 있으면 `value={name ?? ""}`처럼 문자열을 유지한다.
- `defaultValue`를 나중에 바꿔도 현재 DOM 값은 바뀌지 않는다.
- checkbox의 `value`는 제출값이고, 선택 여부는 `checked`다.
