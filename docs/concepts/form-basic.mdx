# Form 구현

## API reference

- `onSubmit` / `event.preventDefault()`: form 제출 handler에서 기본 페이지 이동을 막고 제출 로직을 실행한다.
- `button type`: 제출 버튼은 `type="submit"`, 제출이 아닌 버튼은 `type="button"`을 명시한다.
- `FormData`: `new FormData(event.currentTarget)`로 `name`이 있는 control 값을 제출 시점에 읽는다.
- `value` / `onChange`: text input은 state 값을 `value`에 넣고 다음 값을 `event.currentTarget.value`에서 읽는다.
- `checkbox checked`: checkbox 선택 여부는 `checked`로 제어하고 다음 값을 `event.currentTarget.checked`에서 읽는다.

## 쓰는 곳

입력값을 React state로 제어하거나, 제출 시점의 form 값을 모아 validation과 저장을 시작할 때 쓴다.

## 바로 쓰는 코드

```tsx
const SignupForm = () => {
  const [name, setName] = useState("");
  const [agree, setAgree] = useState(false);

  const submit: React.FormEventHandler<HTMLFormElement> = (event) => {
    event.preventDefault();

    const data = new FormData(event.currentTarget);
    console.log({ name: data.get("name"), agree: data.get("agree") === "yes" });
  };

  return (
    <form onSubmit={submit}>
      <input
        name="name"
        value={name}
        onChange={(event) => setName(event.currentTarget.value)}
      />
      <label>
        <input
          type="checkbox"
          name="agree"
          value="yes"
          checked={agree}
          onChange={(event) => setAgree(event.currentTarget.checked)}
        />
        동의합니다
      </label>
      <button type="button">취소</button>
      <button type="submit">저장</button>
    </form>
  );
};
```

## 실수 포인트

- form 안 버튼은 기본값이 submit이므로 일반 버튼에는 `type="button"`을 붙인다.
- `FormData`는 `name`이 없는 control 값을 읽지 못한다.
- checkbox의 `value`는 제출될 문자열이고, 선택 여부는 `checked`로 읽는다.
