# IntersectionObserver

## API reference

`IntersectionObserver`는 target element가 기준 영역과 얼마나 겹치는지 비동기로 알려준다. 생성자는 callback과 options를 받는다.

```tsx title="observer.tsx"
const observer = new IntersectionObserver(
  (entries, observer) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        observer.unobserve(entry.target); // [!code highlight]
      }
    });
  },
  {
    root: document.querySelector("#scroll-root"),
    rootMargin: "160px 0px",
    threshold: [0, 0.5, 1],
  },
);
```

- `callback(entries, observer)`: 관찰 결과가 생기면 실행된다.
- `entries`: target별 관찰 결과다. 보통 `isIntersecting`, `intersectionRatio`, `target`만 읽는다.
- `observe(target)`: target 관찰을 시작한다.
- `unobserve(target)`: 특정 target만 관찰 대상에서 뺀다.
- `disconnect()`: observer가 보던 모든 target을 해제한다.
- `root`: 기준 영역이다. 생략하거나 `null`이면 viewport 기준이다.
- `rootMargin`: 기준 영역을 넓히거나 줄이는 CSS margin 형식 값이다.
- `threshold`: callback을 실행할 노출 비율이다. 숫자 하나 또는 숫자 배열을 쓴다.

## 쓰는 곳

`IntersectionObserver`는 element가 viewport나 scroll container 안에 들어왔는지 감지할 때 쓴다. `scroll event`를 직접 듣고 매번 좌표를 계산하는 방식보다 단순하고, 브라우저가 관찰 시점을 조절할 수 있다.

- infinite scroll의 끝 sentinel 감지
- 이미지 lazy loading
- 노출 analytics
- 화면 밖으로 나간 popover trigger 처리

## 바로 쓰는 코드

React에서는 ref가 잡힌 뒤 observe하고, effect cleanup에서 observer를 끊는다. target이 조건부 렌더링으로 바뀌면 새 node를 다시 observe한다.

```tsx title="useVisibility.tsx"
const useVisibility = (targetRef: RefObject<Element>) => {
  useEffect(() => {
    const target = targetRef.current;

    if (!target) {
      return;
    }

    const observer = new IntersectionObserver(([entry]) => {
      if (entry?.isIntersecting) {
        // ...
      }
    });

    observer.observe(target); // [!code highlight]

    return () => {
      observer.disconnect(); // [!code highlight]
    };
  }, [targetRef]);
};
```

## 실수 포인트

:::warning
`IntersectionObserver` callback은 React state 변경 직후 동기적으로 실행되는 이벤트 핸들러가 아니다. 요청, state, cleanup을 함께 다룰 때는 중복 실행과 stale closure를 먼저 막아야 한다.
:::

- `root`가 scroll container라면 container element를 넘긴다.
- `rootMargin` 양수는 더 일찍, 음수는 더 안쪽에서 감지한다.
- `threshold` 배열은 callback 빈도를 늘리므로 단계별 analytics에만 쓴다.
- 관찰 target이 바뀌면 기존 observer가 새 node를 자동으로 따라가지 않는다.
- infinite scroll은 `isLoading`과 `hasMore` guard를 callback 진입 초기에 둔다.
