# Virtualized List

## API reference

- `scrollTop`: scroll container 안에서 위로 지나간 높이다.
- `itemHeight`: row 하나의 고정 높이다.
- `viewportHeight`: 화면에 실제로 보이는 list 높이다.
- `overscan`: viewport 위아래로 미리 렌더링할 row 수다.
- `spacer`: 렌더링하지 않은 row의 높이를 빈 영역으로 보존한다.

## 쓰는 곳

- 수백~수만 row 목록에서 DOM node 수를 줄일 때 쓴다.
- row 높이가 고정되어 `scrollTop / itemHeight`로 index를 계산할 수 있을 때 단순하다.

## 바로 쓰는 코드

```ts title="virtualized-list-range.ts"
const getVirtualRange = ({
  itemCount,
  itemHeight,
  overscan,
  scrollTop,
  viewportHeight,
}: {
  itemCount: number;
  itemHeight: number;
  overscan: number;
  scrollTop: number;
  viewportHeight: number;
}) => {
  const visibleCount = Math.ceil(viewportHeight / itemHeight);
  const firstVisibleIndex = Math.floor(scrollTop / itemHeight);
  const startIndex = Math.max(0, firstVisibleIndex - overscan);
  const endIndex = Math.min(
    itemCount,
    firstVisibleIndex + visibleCount + overscan,
  );

  return {
    startIndex,
    endIndex,
    topSpacerHeight: startIndex * itemHeight, // [!code highlight]
    bottomSpacerHeight: (itemCount - endIndex) * itemHeight, // [!code highlight]
  };
};
```

`rows.slice(startIndex, endIndex)`만 렌더링한다. `topSpacerHeight`를 visible row 앞에, `bottomSpacerHeight`를 visible row 뒤에 빈 `div` 높이로 넣으면 전체 scroll 높이는 유지되고 DOM에는 일부 row만 남는다.

## 실수 포인트

- 전체 row를 숨겨 두는 방식은 DOM node 수가 그대로라 virtualization이 아니다.
- 동적 row 높이는 별도 측정 전략이 필요하다.
- `overscan`이 너무 작으면 빠른 스크롤에서 빈 영역이 보이고, 너무 크면 DOM 절감 효과가 줄어든다.
